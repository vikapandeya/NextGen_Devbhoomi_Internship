Beginner Level — 1 Month (4 Weeks)

BA Fundamentals · User Stories · Agile & Scrum · JIRA · Sprint Simulation

Beginner
Learning Objectives
  • Understand the role of a Business Analyst in SDLC and Agile projects
  • Identify and categorize stakeholders using RACI matrix
  • Write clear user stories with acceptance criteria
  • Apply elicitation techniques (interviews, workshops, observation)
  • Understand and practice Scrum ceremonies
  • Manage a JIRA project with Scrum board, backlog, and sprints
Weekly Curriculum
WeekTopicsHands-on LabAssignment
Week 1 Business Analysis fundamentals: BA roles and responsibilities in Waterfall vs Agile projects; BABOK v3 knowledge areas overview; types of requirements (business, stakeholder, functional, non-functional, transition); SDLC phases and BA touchpoints; stakeholder identification (internal/external/primary/secondary); RACI matrix (Responsible, Accountable, Consulted, Informed); power/interest grid for stakeholder prioritization Identify all stakeholders for an e-commerce platform project + build RACI matrix Stakeholder register document with stakeholder analysis for a banking app project
Week 2 Elicitation techniques: structured interviews (open/closed questions, funnel technique), workshops (facilitation, agenda, outcomes), surveys and questionnaires, document analysis, observation (shadowing), prototyping as elicitation; functional vs non-functional requirements (FURPS+ model); user stories format "As a [role], I want [feature], So that [benefit]"; acceptance criteria (Given-When-Then, bullet criteria, rules-based); epics, stories, and tasks hierarchy Conduct a 10-minute requirements interview (role-play) and extract 15 user stories Write 25 user stories with acceptance criteria for a banking mobile app (5 epics)
Week 3 Agile Manifesto (4 values, 12 principles); Agile vs Waterfall comparison (when to use each); Scrum framework: 3 roles (Product Owner, Scrum Master, Development Team), 3 artifacts (Product Backlog, Sprint Backlog, Increment), 4 ceremonies (Sprint Planning, Daily Standup, Sprint Review, Sprint Retrospective); story points and estimation (Fibonacci sequence, planning poker); velocity calculation; sprint burndown chart Simulate Sprint Planning for a 2-week sprint: select stories, assign story points, create sprint goal Full sprint planning document (sprint goal, selected stories, team capacity, velocity target)
Week 4 JIRA Software setup: project creation (Scrum board), epic/story/task/sub-task hierarchy, sprint creation, backlog refinement, story point assignment, active sprint board management, burndown chart reading, JIRA reports (velocity, burndown, sprint); backlog prioritization techniques (MoSCoW, Kano, RICE); Definition of Done vs Acceptance Criteria Set up JIRA Scrum project, populate backlog with 30 stories, run a 2-week sprint simulation Complete JIRA project URL + sprint retrospective document + burndown chart analysis

Intermediate Level — 3 Months (12 Weeks)

Advanced BA Techniques · BPMN · Risk Management · Reporting · Agile Scaling

Intermediate
Month 1 — Advanced BA Techniques & Process Modeling
WeekTopicsLabAssignment
W1BPMN 2.0 process modeling: events (start/intermediate/end), tasks (manual/service/user/script), gateways (exclusive/parallel/inclusive/event-based), sequence flows, pools and lanes, data objects; AS-IS process mapping (current state); TO-BE process design (future state); gap analysisModel the AS-IS order fulfillment process for a retail company in draw.ioAS-IS → TO-BE transformation with gap analysis for a loan approval process
W2Feasibility analysis: technical feasibility, financial feasibility (cost-benefit analysis, ROI, payback period, NPV basics), operational feasibility, legal feasibility; effort estimation techniques (Function Point Analysis, T-shirt sizing, three-point estimation); business case document structureBuild a cost-benefit analysis spreadsheet for a digital transformation initiativeBusiness case document for implementing a CRM system (5-page report)
W3Non-functional requirements deep dive: performance (response time, throughput), scalability (horizontal/vertical), availability (SLA: 99.9% = 8.76 hours downtime/year), security (authentication, authorization, data encryption), usability (accessibility), reliability (MTBF, MTTR); non-functional requirements traceability; testability of NFRsDefine NFRs for a payment gateway system with measurable acceptance criteriaNFR specification document for a healthcare patient portal (HIPAA compliance NFRs included)
W4BDD (Behavior-Driven Development): Given-When-Then specifications; three amigos (BA + Dev + QA) sessions; feature files and acceptance test collaboration; user story mapping (Jeff Patton method): backbone, walking skeleton, story slicing for MVP vs full release; impact mappingUser story map for an e-learning platform (identify MVP vs Phase 2 scope)Story map + BDD scenarios for 3 user journeys + three amigos session notes
Month 2 — Project Management & Agile Scaling
WeekTopicsLabAssignment
W5Project management fundamentals: project charter (objectives, scope, stakeholders, budget, timeline, risks); scope management and scope creep prevention; WBS (Work Breakdown Structure) creation; project schedule (Gantt chart with dependencies); critical path method (CPM); project baselinesCreate a project charter + WBS + Gantt chart for a mobile app development projectFull project plan for a 3-month software project (charter, WBS, Gantt, RACI)
W6Risk management: risk identification (brainstorming, checklist, SWOT, expert judgment); risk analysis (probability-impact matrix, risk score = P × I); risk register (risk ID, description, probability, impact, risk score, response strategy); risk response strategies (avoid, mitigate, transfer, accept); risk monitoring and controlIdentify and assess 15 risks for a cloud migration projectRisk register with top 10 risks + risk response plan for a banking system upgrade
W7SAFe (Scaled Agile Framework) overview: SAFe 6.0 levels (Essential, Large Solution, Portfolio, Full), Agile Release Train (ART), PI (Program Increment) Planning, Features, Enablers; Kanban: workflow visualization, WIP limits, pull system, flow metrics (cycle time, lead time, throughput, cumulative flow diagram); DevOps integration with AgileDesign a Kanban board with 5 swim lanes and WIP limits for a support teamSAFe PI Planning simulation: prepare Team PI Objectives and team backlog for a PI
W8Vendor management: RFP (Request for Proposal) writing, vendor evaluation criteria, contract types (T&M, Fixed-Price, Retainer); SLA management; Change Control Board (CCB), change request process; stakeholder communication plan (RACI + cadence + channels); executive reporting (one-pager, dashboard, steering committee presentations)Write an RFP for a CRM implementation project with evaluation scorecardStakeholder communication plan + monthly executive report template for a digital project
Month 3 — Reporting, Analytics & Capstone
WeekTopicsLabAssignment
W9SQL for Business Analysts: SELECT queries for business reporting, JOINs for combining data sources, GROUP BY for aggregations, subqueries for comparisons, date functions, CASE statements for categorization; connecting to live database and extracting insights20 SQL queries on a business database: revenue analysis, customer cohorts, product performanceBusiness intelligence report using SQL: 10 KPI calculations with findings and recommendations
W10Power BI for BA: data import (SQL/Excel/CSV), Power Query (data transformation, merging, unpivoting), DAX basics (CALCULATE, SUM, AVERAGE, COUNTROWS, IF, DIVIDE, RELATED), creating measures, visualization types (bar/line/map/funnel/KPI card/matrix); dashboard storytelling; publish to Power BI Service + shareBuild a project health dashboard in Power BI (KPIs: velocity, burndown, budget, risks)Executive project status dashboard in Power BI with 5 pages + narrated insights
W11Change management: ADKAR model (Awareness, Desire, Knowledge, Ability, Reinforcement); change impact assessment; training needs analysis; transition planning; benefits realization; OKR framework (Objectives and Key Results) for project alignment; KPIs vs OKRs; product metrics (North Star Metric, HEART framework)Change impact assessment for a legacy system replacement projectOKR framework for a product team + benefit realization plan for a completed project
W12Capstone: Full digital transformation BA engagement — requirements elicitation → BPMN process maps → BRD → risk register → Power BI dashboard → stakeholder presentationConduct elicitation sessions with team members playing stakeholder rolesFull BA engagement deliverable package: 20-page BRD + process maps + dashboard + 15-slide presentation

Enterprise Architecture · Product Management · Data-Driven BA · Program Management · AI in PM

Advanced
MonthFocusTopicsDeliverable

Sample Viva Questions

Beginner Viva

  • What is the difference between a BRD and an FRS?
  • Write a user story for a "forgot password" feature.
  • What are the 4 Agile Manifesto values?
  • What is the purpose of a daily standup?

Intermediate Viva

  • How do you handle a stakeholder who keeps changing requirements?
  • What is the difference between a use case diagram and a BPMN diagram?
  • Explain velocity and how it helps with sprint planning.
  • What is a risk register and how do you prioritize risks?

Advanced Viva

  • How do you prioritize features for a product roadmap when resources are limited?
  • Explain the difference between program management and project management.
  • How would you apply TOGAF concepts to a digital transformation initiative?
  • Walk me through how you would conduct a full PI Planning session for 5 teams.
