Beginner Level — 1 Month (4 Weeks)

Manual Testing · STLC · Test Case Design · Bug Reporting · JIRA · API Testing Basics

Beginner
Learning Objectives
  • Understand SDLC, STLC, and the bug lifecycle
  • Write comprehensive test plans, test cases, and test scenarios
  • Apply test case design techniques (BVA, EP, decision tables)
  • Log, track, and manage defects using JIRA
  • Test REST APIs using Postman with assertions
  • Prepare a complete test summary report
Weekly Curriculum
WeekTopicsHands-on LabAssignment
Week 1 SDLC (Waterfall, Agile, V-model); STLC (6 phases: requirement analysis → test planning → test design → env setup → test execution → closure); Bug lifecycle (New → Assigned → Open → Fixed → Retest → Closed/Reopened); Types of testing (functional, regression, smoke, sanity, UAT, exploratory, ad-hoc); Severity vs Priority matrix Write 20 test cases for a login module (positive + negative + edge cases) Test plan document for an e-commerce website (scope, in-scope/out-of-scope, risks, schedule)
Week 2 Test case design techniques: Equivalence Partitioning (EP), Boundary Value Analysis (BVA), Decision Table Testing, State Transition Testing, Error Guessing; Test case writing best practices (steps, expected result, actual result, status, priority); Test data management Apply all 5 techniques to a registration form with 10 input fields 50 test cases for a registration page using EP + BVA + decision table (no duplicates)
Week 3 Bug report writing: title (clear, concise, unique), description, steps to reproduce, expected vs actual result, severity/priority, environment, attachments (screenshots/video); JIRA setup: project creation, issue types, boards; Agile QA process: sprint-based testing, definition of done; TestRail: test suites, test runs, milestones Log 10 real bugs found in OpenCart demo site in JIRA with all fields Create a complete test cycle in TestRail for a login module + execute and report results
Week 4 API fundamentals: REST, HTTP methods, status codes (2xx/3xx/4xx/5xx), JSON/XML; Postman: collections, environments, variables, pre-request scripts, pm.test() assertions, response body/status/time validation; Newman CLI for running collections Test Reqres.in API: write 15 assertions covering all CRUD operations Full Postman collection for PetStore API + run with Newman + HTML report

Sample Viva Questions

Beginner Viva

  • What is the difference between Severity and Priority in bug reporting?
  • Explain Boundary Value Analysis with an example.
  • What is regression testing and when do you perform it?
  • What HTTP status code means "Not Found"?

Intermediate Viva

  • What is the Page Object Model and why use it?
  • Explain the difference between implicit wait and explicit wait in Selenium.
  • What is the difference between load testing and stress testing?
  • How does BDD benefit both developers and business stakeholders?

Advanced Viva

  • How do you design a test strategy for a microservices application?
  • What is contract testing and when does it replace integration testing?
  • How would you set up a zero-downtime deployment QA process?
  • Calculate defect leakage rate for a sprint with 20 bugs found post-release.
