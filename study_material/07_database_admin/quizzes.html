Database Administration Quizzes

Five weekly knowledge checks covering database basics, SQL queries, schema design, indexing, and transactions. Complete each week's content before attempting the quiz.

Week 1 — Database Basics

5 Questions

1. ACID stands for:

ACID ensures reliable database transactions even in failure scenarios: Atomicity (all-or-nothing), Consistency (valid state), Isolation (concurrent safety), Durability (committed data persists).

2. A PRIMARY KEY must be:

Primary key uniquely identifies each row and cannot be NULL. It can be any data type — strings (ISBN, email) are valid primary keys when they are naturally unique.

3. Which SQL statement is used to retrieve data from a table?

SELECT is DML (Data Manipulation Language) used to query/retrieve data. INSERT adds rows, UPDATE modifies rows, CREATE is DDL that defines structure.

4. A FOREIGN KEY constraint ensures:

Foreign key prevents orphaned rows — you cannot insert an order with customer_id=999 if no customer with id=999 exists. This enforces referential integrity.

5. Running DELETE without a WHERE clause:

DELETE without WHERE removes every row (but keeps the table structure). Always run a SELECT with the same WHERE first to verify what you are about to delete. Use TRUNCATE to quickly delete all rows.

Week 2 — SQL Queries

5 Questions

1. HAVING is used to filter:

WHERE filters rows before grouping. HAVING filters groups after GROUP BY. You cannot use aggregate functions (COUNT, SUM) in WHERE — use HAVING instead.

2. SELECT COUNT(*) FROM orders WHERE status='pending' returns:

COUNT(*) returns a single integer — the count of rows matching the WHERE condition. To get total amount, use SUM(total) instead of COUNT(*).

3. LEFT JOIN returns:

LEFT JOIN: every row from the left table appears in the result. If no matching row exists in the right table, the right table columns show NULL. Useful for finding records without a match (customers with no orders).

4. COALESCE(phone, 'Not Provided') returns:

COALESCE returns the first non-NULL argument. It is essential for replacing NULLs with meaningful defaults in reports and user-facing output.

5. ORDER BY price DESC LIMIT 5 returns:

DESC = descending = highest first. So ORDER BY price DESC gives the most expensive at the top. LIMIT 5 keeps only the first 5 rows of that sorted result.

Week 3 — Database Design

5 Questions

1. Third Normal Form (3NF) eliminates:

3NF removes transitive dependencies: if A → B → C (non-key B determines non-key C), then B and C should be in a separate table. Example: zip_code → city → state should be in a zip_codes lookup table.

2. A one-to-many relationship between departments and employees means:

1:N (one-to-many): the "one" side (department) has a primary key, the "many" side (employees) has a foreign key pointing to the department. Each employee belongs to one department; each department has many employees.

3. Denormalisation in a database is done primarily to:

Analytics and reporting databases (data warehouses) often use denormalised star/snowflake schemas — storing pre-computed aggregates or repeated data to avoid expensive JOINs on every query.

4. Which data type is best for storing currency amounts?

FLOAT and DOUBLE use binary floating point, which cannot exactly represent most decimal fractions — 0.1 + 0.2 ≠ 0.3 in binary. DECIMAL stores exact values and is mandatory for financial data.

5. An ER (Entity-Relationship) diagram shows:

An ER diagram is the visual blueprint of a database before writing SQL. It shows entities, attributes, and the cardinality of relationships (1:1, 1:N, M:N).

Week 4 — Indexing

5 Questions

1. An index primarily improves:

Indexes dramatically speed up SELECT queries at the cost of slightly slower writes (the index must also be updated) and additional storage. This trade-off is almost always worth it for frequently-queried columns.

2. EXPLAIN shows type=ALL for a query. This means:

type=ALL is the worst case — MySQL reads every single row in the table. For large tables, this is very slow. Adding an appropriate index changes this to type=ref or type=range.

3. A composite index on (col1, col2) helps queries that filter on:

The "leftmost prefix" rule: a composite index (A, B, C) helps queries filtering on A, or A+B, or A+B+C. It does NOT help queries filtering on B alone, C alone, or B+C without A.

4. Which column is LEAST suitable for indexing?

Low-cardinality columns (is_deleted, gender, boolean flags) make poor index candidates. When half the rows match WHERE is_deleted=0, a full table scan is often faster than using an index and then fetching half the table's rows.

5. A UNIQUE INDEX prevents:

UNIQUE INDEX enforces uniqueness + provides fast lookups. Note: unlike PRIMARY KEY, UNIQUE allows NULL values (and multiple NULLs are permitted since NULL ≠ NULL).

Week 5 — Transactions

5 Questions

1. If a transaction fails midway and you run ROLLBACK:

ROLLBACK undoes ALL changes made within the current transaction — this is the Atomicity guarantee of ACID. The database returns to its state exactly as it was before BEGIN/START TRANSACTION.

2. The READ COMMITTED isolation level prevents:

Dirty read: reading data that another transaction has modified but not yet committed — that transaction might ROLLBACK, making your read invalid. READ COMMITTED prevents this by only reading committed data.

3. A database deadlock occurs when:

Deadlock prevention: always acquire locks in the same order across all transactions. MySQL detects deadlocks automatically and kills the lower-cost transaction (victim), which your application should catch and retry.

4. BEGIN / START TRANSACTION starts:

Without an explicit transaction, MySQL runs in autocommit mode — each statement commits immediately. BEGIN disables autocommit for that session until COMMIT or ROLLBACK.

5. Connection pooling in a database-backed application is used to:

Creating a new DB connection per request takes 50–200ms and is expensive. A pool of 10 pre-warmed connections handles 10 concurrent requests instantly. Most Node.js DB libraries (mysql2, pg) provide built-in pool support.
