Week 1 Quiz — Linux Command Line

Week 1 — Linux Foundations

5 Questions · Beginner

Question 1 of 5

Which command shows your current directory in Linux?

A) dir
B) pwd
C) where
D) location
pwd = print working directory. It shows the full absolute path of where you are in the filesystem, e.g. /home/ubuntu/projects. Run it whenever you're lost in the terminal.

Question 2 of 5

What permission does chmod 755 give to the owner?

A) Read only
B) Read + Write
C) Read + Write + Execute
D) No permission
7 = 4+2+1 = rwx for the owner (full access). The second digit 5 = 4+0+1 = r-x for group, and the third 5 = r-x for others. So 755 means owner can do everything; everyone else can read and execute but not write.

Question 3 of 5

Which command searches for text inside files?

A) find
B) locate
C) grep
D) search
grep searches file content with regex. find searches file names. Example: grep "error" /var/log/syslog — or pipe it: cat access.log | grep "500" to find all HTTP 500 errors.

Question 4 of 5

What does sudo mean?

A) Super User Desktop Operations
B) Switch User and Do
C) Run command as superuser/root
D) Secure Unix Directory Operations
sudo (substitute user do) runs commands with elevated root privileges. Always use sparingly — sudo rm -rf / would delete your entire system. In production, avoid running applications as root.

Question 5 of 5

Which command lists all running processes?

A) list
B) jobs
C) ls -p
D) ps aux
ps aux shows all processes with user, CPU%, memory%, PID, and command name. Pipe with grep to filter: ps aux | grep nginx. Use kill -9 PID to forcefully stop a process by its ID.
Week 2 Quiz — Git & Version Control

Week 2 — Git Version Control

5 Questions · Beginner

Question 1 of 5

What is the correct Git workflow order?

A) commit → add → push
B) add → commit → push
C) push → commit → add
D) commit → push → add
Stage → Snapshot → Upload: git add stages changes (picks what to include), git commit snapshots them with a message, git push uploads to the remote. You cannot push before committing.

Question 2 of 5

What does git pull do?

A) Uploads your code
B) Downloads + merges latest changes from remote
C) Creates a new branch
D) Deletes a remote branch
git pull = git fetch + git merge. It downloads the latest commits from the remote and merges them into your current branch. Always pull before starting new work to avoid merge conflicts.

Question 3 of 5

What is a Pull Request (PR)?

A) Downloading code from a repository
B) Requesting someone to review and merge your branch
C) Pulling the latest tags from remote
D) A type of Git conflict
A Pull Request is a GitHub feature (not a Git command) where you ask teammates to review your branch before merging into main. This is the core of professional team collaboration — no one pushes directly to main.

Question 4 of 5

Which file tells Git which files to ignore?

A) .gitskip
B) .gitexclude
C) .gitignore
D) ignore.git
.gitignore lists patterns of files/folders Git should never track. Every project needs one. Common entries: node_modules/, .env, *.log, .DS_Store. Create it before your first commit.

Question 5 of 5

What happens if you commit a .env file with secrets to a public GitHub repo?

A) Nothing — GitHub is private by default
B) Git refuses the commit
C) Your secrets are exposed publicly and must be immediately rotated
D) GitHub auto-encrypts them
Exposed secrets are a real incident. Even if you delete the file and push again, the secrets remain in Git history. Bots scan GitHub within seconds of a push. Always add .env to .gitignore before your first commit, and use GitHub Secrets for CI/CD.
Week 3 Quiz — Networking Fundamentals

Week 3 — Networking for DevOps

5 Questions · Beginner

Question 1 of 5

What port does HTTPS use?

A) 80
B) 8080
C) 21
D) 443
Port 443 = HTTPS (encrypted). Port 80 = HTTP (unencrypted). Port 22 = SSH. Port 21 = FTP. Port 3306 = MySQL. Memorise these — you'll configure them in AWS Security Groups constantly.

Question 2 of 5

What does DNS do?

A) Encrypts web traffic
B) Translates domain names to IP addresses
C) Assigns IP addresses to devices
D) Monitors network speed
DNS = Domain Name System — the internet's phonebook. When you type google.com, DNS resolves it to an IP like 142.250.195.46. DHCP assigns IPs; TLS/SSL encrypts traffic; DNS only resolves names.

Question 3 of 5

Which is a private IP address range?

A) 8.8.8.8
B) 142.250.x.x
C) 192.168.x.x
D) 1.1.1.1
RFC 1918 private ranges: 192.168.x.x, 10.x.x.x, and 172.16–31.x.x. They are not routable on the public internet — your home router and AWS VPC both use these. 8.8.8.8 is Google's public DNS; 1.1.1.1 is Cloudflare's.

Question 4 of 5

SSH uses which port by default?

A) 22
B) 443
C) 3306
D) 8080
Port 22 is the default for SSH. In production, restrict SSH access in your Security Group to your own IP only — never open port 22 to 0.0.0.0/0 on a production server. Many teams also change it to a non-standard port.

Question 5 of 5

What does a Security Group in AWS act as?

A) User management system
B) Database firewall
C) Virtual stateful firewall for EC2 instances
D) Load balancer
An AWS Security Group is a stateful virtual firewall. You define inbound and outbound rules by port, protocol, and source IP/CIDR. "Stateful" means if you allow inbound traffic, the return response is automatically allowed without a separate outbound rule.
Week 4 Quiz — Cloud Computing Intro

Week 4 — Cloud Concepts & AWS Basics

5 Questions · Beginner

Question 1 of 5

What is IaaS?

A) Software delivered via the internet
B) A platform for deploying apps
C) Raw infrastructure (VMs, storage, networking) you manage
D) Internet as a Service
IaaS (Infrastructure as a Service) gives you raw compute, storage, and networking — you manage the OS up. PaaS handles the runtime too (e.g. Heroku). SaaS is the finished app (e.g. Gmail). AWS EC2 is IaaS.

Question 2 of 5

Which AWS service stores static files like images, videos, and HTML?

A) EC2
B) RDS
C) Lambda
D) S3
S3 (Simple Storage Service) stores objects (files) at 99.999999999% durability. EC2 is for virtual servers; RDS is managed databases; Lambda is serverless functions. S3 also hosts static websites directly.

Question 3 of 5

What is the AWS region code for Mumbai, India?

A) ap-southeast-1
B) ap-south-1
C) us-east-1
D) eu-west-1
ap-south-1 is AWS Mumbai — the lowest latency region for Indian users. ap-southeast-1 is Singapore. Always select ap-south-1 for Indian projects unless there's a specific reason (e.g. disaster recovery in a second region).

Question 4 of 5

What should you set immediately after creating a new AWS account?

A) Disable MFA for convenience
B) A billing alarm
C) Delete the default VPC
D) Enable root user CLI access
Set a billing alarm in CloudWatch immediately. This alerts you if spending exceeds your threshold (e.g. ₹500). AWS charges for everything — a forgotten EC2 instance can rack up thousands in a month. Also enable MFA on root.

Question 5 of 5

What is the AWS Free Tier?

A) Paid tier for enterprise accounts
B) Permanent free services + 12-month free tier for new accounts
C) Free for students only
D) Free forever for all services
AWS Free Tier has three types: Always free (Lambda 1M requests/month, DynamoDB 25GB), 12 months free after signup (EC2 t2.micro 750 hrs/month, S3 5GB), and trials (short-term free for specific services). Monitor usage carefully — it expires!
Week 5 Quiz — Docker & Containers

Week 5 — Docker & Containerisation

5 Questions · Intermediate

Question 1 of 5

What is a Docker image?

A) A screenshot of a running container
B) A read-only template used to create containers
C) A type of virtual machine
D) Docker's version of a file
A Docker image is a read-only layered template built from a Dockerfile. Running an image creates a container — a live, isolated process. Think of it as: image = recipe, container = the dish being cooked.

Question 2 of 5

Which Dockerfile instruction sets the working directory inside the container?

A) RUN
B) DIR
C) WORKDIR
D) SET
WORKDIR /app sets the current directory for all subsequent RUN, COPY, CMD, and ENTRYPOINT instructions. If the directory doesn't exist, Docker creates it. Always use WORKDIR instead of RUN cd /app.

Question 3 of 5

What does docker-compose up do?

A) Updates Docker to the latest version
B) Builds and starts all services defined in docker-compose.yml
C) Uploads your image to Docker Hub
D) Updates all base images
docker-compose up -d builds (if needed) and starts all services defined in your docker-compose.yml in detached mode. Use it to spin up your entire stack (API + database + cache) with one command.

Question 4 of 5

Why use alpine-based images like node:20-alpine?

A) They have more features
B) Better security certificates
C) Much smaller image size — faster deploys, less attack surface
D) Required for production environments
Alpine Linux is ~5 MB vs Ubuntu's ~75 MB. Smaller images = faster CI/CD pulls, less ECR storage cost, and a reduced attack surface (fewer packages = fewer CVEs). Always prefer -alpine variants unless you need specific glibc tools.

Question 5 of 5

What file should every Docker project include?

A) .dockerenv
B) .dockerignore
C) Containerfile
D) docker.config
.dockerignore prevents node_modules/, .env, .git/, and other unnecessary files from being copied into the image. Without it, a COPY . . instruction will bloat your image with hundreds of MB of dependencies you don't need.
