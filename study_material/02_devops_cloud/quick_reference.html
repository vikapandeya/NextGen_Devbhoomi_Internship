Quick Reference Sheet — DevOps & Cloud Engineering

Essential Linux, Docker, Git, AWS CLI, Terraform, and Kubernetes commands. Print and keep on your desk. Everything you need in one place.

Printable All Levels Keep This Handy
Linux Commands
Navigation & Files
pwd                    # print working directory
ls -la                 # list all files with permissions
cd /path/to/dir        # change directory
cd ..                  # go up one level
mkdir -p dir/subdir    # create nested directories
touch file.txt         # create empty file
cp src.txt dst.txt     # copy file
mv old.txt new.txt     # move / rename file
rm -rf directory/      # delete directory (careful!)
cat file.txt           # print file contents
less file.txt          # scroll through file
head -20 file.txt      # first 20 lines
tail -f /var/log/app   # follow log in real time
Search & Processes
grep -r "text" .       # search all files recursively
grep -n "error" app.log# show line numbers
find . -name "*.log"   # find files by name pattern
ps aux                 # list all running processes
ps aux | grep nginx    # filter processes by name
kill -9 PID            # force kill process by PID
top                    # live process monitor
htop                   # better interactive process view
df -h                  # disk usage (human readable)
free -h                # memory usage
Permissions & Networking
chmod 755 script.sh    # rwx for owner, r-x for others
chmod 600 key.pem      # read-only for owner (SSH keys)
chown user:group file  # change file ownership
sudo command           # run as root
ssh user@hostname      # connect via SSH
ssh -i key.pem ec2-user@ip  # SSH with key file
scp file user@host:/path    # copy file to remote
curl -I https://url    # check HTTP response headers
curl -s url | jq .     # fetch JSON and pretty-print
ping hostname          # test connectivity
nslookup domain.com    # DNS lookup
netstat -tulpn         # show listening ports
System & Package Management
sudo apt update        # refresh package list (Debian/Ubuntu)
sudo apt install nginx # install a package
sudo systemctl start nginx   # start a service
sudo systemctl status nginx  # check service status
sudo systemctl enable nginx  # start on boot
journalctl -u nginx -f  # follow service logs
uname -r               # kernel version
lsb_release -a         # OS version
which docker           # find binary location
export VAR="value"     # set env variable in session
echo $HOME             # print env variable
Docker Commands
Images
docker build -t name:tag .     # build image from Dockerfile
docker build --no-cache -t n:t . # force fresh build
docker images                  # list local images
docker pull nginx:alpine       # pull from Docker Hub
docker push user/name:tag      # push to registry
docker rmi image-id            # remove image
docker image prune             # remove unused images
docker inspect image-id        # full image metadata
Containers
docker run -d -p 80:3000 \
  --name app image:tag         # run detached, port mapped
docker run --rm -it image sh   # interactive, auto-remove
docker ps                      # running containers
docker ps -a                   # all containers (inc. stopped)
docker stop app                # graceful stop
docker rm app                  # remove stopped container
docker logs app                # view container logs
docker logs -f app             # follow logs live
docker exec -it app sh         # open shell in container
docker stats                   # live resource usage
docker-compose
docker-compose up -d           # start all services detached
docker-compose down            # stop + remove containers
docker-compose down -v         # also remove volumes
docker-compose ps              # list service containers
docker-compose logs -f         # follow all service logs
docker-compose logs -f web     # follow one service
docker-compose build           # rebuild all images
docker-compose restart web     # restart one service
docker-compose exec web sh     # shell into service
Dockerfile Essentials
FROM node:20-alpine        # base image
WORKDIR /app               # set working directory
COPY package*.json ./      # copy package files first
RUN npm ci --only=prod     # install production deps
COPY . .                   # copy app source
EXPOSE 3000                # document the port
USER node                  # run as non-root user
CMD ["node", "server.js"]  # start command
Git Commands
Daily Workflow
git status                 # see changed files
git diff                   # see unstaged changes
git add .                  # stage all changes
git add file.js            # stage specific file
git commit -m "feat: ..."  # commit with message
git push                   # push to remote
git pull                   # pull latest from remote
git log --oneline -10      # compact recent history
git show abc123            # inspect a commit
Branching
git checkout -b feature/name  # create + switch branch
git checkout main          # switch to main
git branch                 # list local branches
git branch -a              # list all (inc. remote)
git merge feature/name     # merge into current branch
git branch -d feature/name # delete merged branch
git push origin --delete f # delete remote branch
git stash                  # save work-in-progress
git stash pop              # restore stashed changes
Advanced & Recovery
git revert HEAD            # undo last commit safely
git reset HEAD~1           # undo commit, keep changes
git reset --hard HEAD~1    # undo commit, discard changes
git cherry-pick abc123     # apply one commit to branch
git tag v1.0.0             # create a version tag
git tag -a v1.0.0 -m "..."  # annotated tag
git push --tags            # push all tags
git blame file.js          # who changed each line?
git bisect start           # binary search for a bug

AWS CLI Commands
Config & S3
aws configure              # set up credentials + region
aws configure list         # show current config
aws s3 ls                  # list all buckets
aws s3 ls s3://bucket-name # list bucket contents
aws s3 mb s3://bucket-name # create bucket
aws s3 cp file.txt s3://bucket/  # upload file
aws s3 sync ./dist s3://bucket/  # sync directory
aws s3 rb s3://bucket --force    # delete bucket + contents
EC2
aws ec2 describe-instances  # list all EC2 instances
aws ec2 describe-instances \
  --query 'Reservations[].Instances[].
    [InstanceId,State.Name,PublicIpAddress]' \
  --output table            # formatted table view
aws ec2 start-instances \
  --instance-ids i-xxxxxxxx # start stopped instance
aws ec2 stop-instances \
  --instance-ids i-xxxxxxxx # stop instance
aws ec2 describe-security-groups  # list security groups
ECR & ECS
# Authenticate Docker to ECR
aws ecr get-login-password \
  --region ap-south-1 | docker login \
  --username AWS \
  --password-stdin <account>.dkr.ecr.ap-south-1.amazonaws.com

aws ecr create-repository \
  --repository-name myapp  # create ECR repo
aws ecr describe-repositories  # list repos
aws ecs list-clusters      # list ECS clusters
aws ecs describe-services \
  --cluster name --services svc # service details
IAM & Useful Flags

aws iam list-users         # list all IAM users
aws sts get-caller-identity  # who am I? (account + ARN)

--dry-run                  # test without making changes

HCL — Provider & Resource
provider "aws" {
  region = var.aws_region
}

resource "aws_s3_bucket" "main" {
  bucket = var.bucket_name
  tags = {
    Environment = var.env
    Project     = "ndfi-devops"
  }
}

resource "aws_instance" "web" {
  ami           = data.aws_ami.ubuntu.id
  instance_type = "t2.micro"
  vpc_security_group_ids = [aws_security_group.web.id]
}
HCL — Variables & Outputs
variable "bucket_name" {
  type        = string
  description = "Name of the S3 bucket"
  default     = "ndfi-assets"
}

variable "env" {
  type    = string
  validation {
    condition     = contains(["dev","prod"], var.env)
    error_message = "Must be dev or prod."
  }
}

output "bucket_arn" {
  value       = aws_s3_bucket.main.arn
  description = "ARN of the created bucket"
}
Kubernetes (kubectl)
Get & Describe Resources
kubectl get pods                    # list pods (default namespace)
kubectl get pods -n kube-system     # specific namespace
kubectl get pods -A                 # all namespaces
kubectl get svc                     # list services
kubectl get deployments             # list deployments
kubectl get nodes                   # list cluster nodes
kubectl get all                     # all resource types
kubectl describe pod pod-name       # detailed pod info
kubectl describe node node-name     # node details + conditions
Deploy & Scale
kubectl apply -f manifest.yaml      # create / update resources
kubectl apply -f k8s/               # apply whole directory
kubectl delete -f manifest.yaml     # delete resources from file
kubectl delete pod pod-name         # delete a pod (restarts!)
kubectl scale deployment app \
  --replicas=3                      # scale deployment
kubectl rollout restart \
  deployment/app                    # rolling restart
kubectl rollout status \
  deployment/app                    # check rollout progress
kubectl rollout undo deployment/app # rollback to previous
Debug & Logs
kubectl logs pod-name               # pod logs
kubectl logs -f pod-name            # follow logs live
kubectl logs pod-name -c container  # multi-container pod
kubectl exec -it pod-name -- sh     # shell into pod
kubectl port-forward \
  svc/app 8080:80                   # forward service port locally
kubectl get events \
  --sort-by='.lastTimestamp'        # cluster events
kubectl top pods                    # CPU + memory per pod
kubectl top nodes                   # CPU + memory per node
Manifest — Deployment + Service
apiVersion: apps/v1
kind: Deployment
metadata:
  name: api
spec:
  replicas: 2
  selector:
    matchLabels: { app: api }
  template:
    metadata:
      labels: { app: api }
    spec:
      containers:
      - name: api
        image: <ecr-url>/api:latest
        ports: [{containerPort: 3000}]
---
apiVersion: v1
kind: Service
metadata: { name: api }
spec:
  selector: { app: api }
  ports: [{port: 80, targetPort: 3000}]
GitHub Actions — CI/CD Snippet
Complete: Test → Build → Push to ECR → Deploy to EC2
# .github/workflows/deploy.yml
name: Deploy
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with: { node-version: '20' }

      - name: Test
        run: npm ci && npm test

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ap-south-1

      - name: Login to ECR
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v2

      - name: Build and push image
        run: |
          IMAGE=${{ steps.login-ecr.outputs.registry }}/myapp:${{ github.sha }}
          docker build -t $IMAGE .
          docker push $IMAGE
          echo "IMAGE=$IMAGE" >> $GITHUB_ENV

      - name: Deploy to EC2
        run: |
          echo "${{ secrets.EC2_SSH_KEY }}" > key.pem
          chmod 600 key.pem
          ssh -i key.pem -o StrictHostKeyChecking=no \
            ec2-user@${{ secrets.EC2_HOST }} \
            "docker pull $IMAGE && docker stop app; docker rm app; \
             docker run -d --name app -p 80:3000 $IMAGE"
