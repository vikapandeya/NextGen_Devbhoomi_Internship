Project Briefs — Mobile App Development

Three graded projects that grow from beginner to advanced. Each has a full feature specification and a transparent rubric.

P1
Personal Habit Tracker
Beginner
React Native (Expo) AsyncStorage React Navigation expo-notifications

An app to track daily personal habits — drinking water, exercising, reading, and more. The user adds their own habits, marks completion each day, and sees their current streak. All data is stored locally on the device — no backend required.

Requirements

  • Add and delete custom habits (name + emoji icon)
  • Mark a habit as done for today with a single tap (toggle)
  • Streak counter: counts consecutive days with all habits completed
  • Weekly view: 7-day calendar row showing ✓ / ✗ for each habit
  • Local notification at a user-set time using expo-notifications
  • All data persisted in AsyncStorage — survives app restarts and phone restarts
  • Minimum 3 screens: Home (today's habits), History (weekly view), Settings (add/delete habits, notification time)
  • Clean UI: each habit displayed as a card with color, emoji, name, and today's completion status

P2
Recipe Finder App
Intermediate
React Native Expo React Navigation TheMealDB API expo-image-picker AsyncStorage

Search and browse recipes from a real public API. Save favourites locally for offline access. Build a shopping list from recipe ingredients, and let users replace recipe photos with their own. This project introduces real-world API integration, proper loading/error state handling, and offline-first design.

Requirements

  • Search recipes by name or ingredient using the real TheMealDB API (themealdb.com/api.php)
  • Browse recipes by category in a grid layout
  • Recipe detail screen: full ingredients list, step-by-step instructions, YouTube video link (open with expo-linking)
  • Favourite recipes: saved in AsyncStorage, accessible offline when there is no internet
  • Shopping list: tap to add all ingredients from any recipe; check off items as you buy them
  • Custom photo: replace the recipe cover image using camera or gallery (expo-image-picker)
  • Pull to refresh on search results
  • Explicit loading and error states on every screen that fetches data

P3
Campfire — Community Discussion App
Advanced
React Native Expo EAS Build React Navigation React Query JWT Auth AsyncStorage Animated API

Build Campfire — a community discussion app with real user authentication, a post feed with infinite scroll, the ability to create posts with images, comments, and push notifications. This project must be submitted as an installable APK built with EAS Build and must include a CI pipeline via GitHub Actions.

Requirements

  • JWT login + registration (against a provided mock API or your own Express backend)
  • Post feed with infinite scroll (FlatList + pagination — load more on scroll-to-end)
  • Create post: text + optional image selected from camera or gallery
  • Animated like button: scale spring effect (Animated.spring) on press
  • Comments: add new comments and delete your own comments
  • User profile screen: avatar, bio, post count, and a grid of the user's posts
  • Local push notifications: "Someone liked your post!"
  • EAS Build: produce an installable APK submitted to Google Play internal testing
  • GitHub Actions CI: lint + test + EAS build triggered on push to main
  • Test coverage ≥ 60% on all custom hooks and reducers
  • Dark mode toggle stored in AsyncStorage and applied app-wide
