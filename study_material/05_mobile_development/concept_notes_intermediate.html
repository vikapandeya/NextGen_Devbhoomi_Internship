Week 5 — State Management & Context API
W5

State Management — Context, useReducer & Zustand

Prop drilling · createContext · useReducer · Zustand basics

▼
The prop drilling problem: Imagine passing a single piece of paper through 6 people in a row just to give it to the person at the end. Each middle person has to handle paper they don't care about. That is prop drilling — and Context API is the internal messaging system that lets you skip the chain.

React Context API

  • createContext(defaultValue) — create the context
  • <AuthContext.Provider value={'{'}user{'}'}> — wrap the tree
  • useContext(AuthContext) — read value anywhere inside the tree
  • Best for: auth state, theme, language, user preferences
  • Does not replace all useState — only global/cross-cutting concerns

useReducer

  • Like Redux but built into React — no extra package
  • const [state, dispatch] = useReducer(reducer, initialState)
  • Dispatch an action: dispatch({'{'}type: 'ADD_TODO', payload: text{'}'})
  • Reducer: pure function (state, action) => newState
  • Use when: multiple sub-values, complex transitions, action-based logic

Zustand — Lightweight State

  • npm install zustand
  • Create a store: const useStore = create((set) => ({'{'}...{'}'}))
  • Read: const count = useStore(s => s.count)
  • Update: set({'{'}count: count + 1{'}'})
  • No Provider, no boilerplate — one file, works everywhere
  • Zustand store can persist to AsyncStorage with a middleware

AsyncStorage + Context Pattern

  • Load from AsyncStorage → set into Context on app start
  • When Context state changes → save to AsyncStorage
  • Result: global state that survives app close
  • Common for: auth token, user settings, theme preference
SituationUseWhy
Local component state (a counter, input value)useStateSimple, isolated, no sharing needed
Complex state with multiple actionsuseReducerPredictable transitions, easy to test
Shared state across many screensContext + useReducerNo prop drilling, standard React pattern
App-wide state, frequent updatesZustandNo re-render overhead, minimal boilerplate
Server/API data with cachingReact QueryHandles loading/error/cache automatically (Week 8)
 
// AuthContext.js — auth state available everywhere
import { createContext, useContext, useReducer, useEffect } from 'react';
import AsyncStorage from '@react-native-async-storage/async-storage';

const AuthContext = createContext(null);

function authReducer(state, action) {
  switch (action.type) {
    case 'LOGIN':  return { user: action.payload, token: action.token };
    case 'LOGOUT': return { user: null, token: null };
    default: return state;
  }
}

export function AuthProvider({ children }) {
  const [state, dispatch] = useReducer(authReducer, { user: null, token: null });

  useEffect(() => {
    async function loadToken() {
      const token = await AsyncStorage.getItem('auth_token');
      if (token) dispatch({ type: 'LOGIN', token });
    }
    loadToken();
  }, []);

  return <AuthContext.Provider value={{'{'}state, dispatch{'}'}>{children}</AuthContext.Provider>;
}

export const useAuth = () => useContext(AuthContext);

Week 5 Key Takeaways

  • Context is not a replacement for all state — only for data that many screens need
  • useReducer makes complex state changes predictable and testable
  • Zustand is the industry favourite for medium-complexity apps — learn its pattern
  • Combine Context with AsyncStorage to get persistent global state with no external library
Week 6 — APIs & Networking
W6

REST API Integration with fetch & Axios

Loading states · Error handling · Axios interceptors · useEffect cleanup

▼
Real-world analogy: Your app is a customer in a restaurant. The REST API is the kitchen. fetch() is the waiter who carries your order and brings the food back. Sometimes the kitchen is busy (loading), sometimes they get your order wrong (error), and sometimes the food arrives perfectly (data). Handle all three states in your UI.

fetch() in React Native

  • Same browser fetch API — works identically in RN
  • Returns a Promise — always use async/await
  • Two steps: await fetch(url) then await res.json()
  • Wrap in try/catch to handle network errors
  • For POST: pass method, headers, body: JSON.stringify(data)

Axios Advantages

  • npm install axios
  • Auto-parses JSON — no .json() step
  • Request/response interceptors — attach auth tokens globally
  • axios.create({'{'}baseURL, timeout{'}'}) — reusable instance
  • Throws on non-2xx status — no manual if(!res.ok) check
  • Cancel requests with AbortController or cancel token

Loading / Error / Data Pattern

  • const [data, setData] = useState(null)
  • const [isLoading, setIsLoading] = useState(true)
  • const [error, setError] = useState(null)
  • Show <ActivityIndicator /> while loading
  • Show error message with retry button on error
  • Show data list when all good

useEffect Cleanup

  • If component unmounts before fetch completes → state update error
  • Solution: AbortController — cancel the request on cleanup
  • Return a cleanup function from useEffect: return () => controller.abort()
  • This prevents "can't update state on unmounted component" warning
import axios from 'axios';
import { ActivityIndicator } from 'react-native';

// Reusable axios instance
const api = axios.create({
  baseURL: 'https://newsapi.org/v2',
  timeout: 8000,
  headers: { Authorization: `Bearer ${API_KEY}` },
});

function NewsScreen() {
  const [articles, setArticles] = useState([]);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const controller = new AbortController();
    async function fetchNews() {
      try {
        const { data } = await api.get('/top-headlines', {
          params: { country: 'in' },
          signal: controller.signal,
        });
        setArticles(data.articles);
      } catch (err) {
        if (!axios.isCancel(err)) setError('Could not load news. Tap to retry.');
      } finally {
        setIsLoading(false);
      }
    }
    fetchNews();
    return () => controller.abort(); // cleanup on unmount
  }, []);

  if (isLoading) return <ActivityIndicator size="large" color="#1B3A6B" />;
  if (error) return <Text onPress={fetchNews}>{error}</Text>;

  return <FlatList data={articles} keyExtractor={a => a.url} renderItem={...} />;
}

Week 6 Key Takeaways

  • Always handle three states: loading, error, and data — never leave the user staring at a blank screen
  • Use Axios for real projects — auto JSON parsing and interceptors save significant boilerplate
  • Use AbortController in useEffect cleanup to prevent memory leaks from cancelled requests
  • Show a retry button on errors — users need a path forward when the network fails
Week 7 — Device Features & Permissions
W7

Expo Modules — Camera, Location, Notifications

expo-image-picker · expo-location · expo-notifications · expo-camera

▼
Expo managed workflow advantage: Accessing the camera on a raw React Native project requires touching native iOS/Android code. Expo modules wrap all of that in a clean JS API. You never write a line of Swift or Kotlin — and you still get full access to the camera, GPS, sensors, and notifications.

expo-image-picker

  • npx expo install expo-image-picker
  • launchImageLibraryAsync() — open photo gallery
  • launchCameraAsync() — open camera to take photo
  • Request permissions first with requestMediaLibraryPermissionsAsync()
  • Returns {'{'}canceled, assets: [{'{'}uri{'}'}, ...]{'}'}
  • allowsEditing: true, aspect: [4,3], quality: 0.8 options

expo-location

  • npx expo install expo-location
  • requestForegroundPermissionsAsync() — ask user for GPS
  • getCurrentPositionAsync({}) — one-time location fix
  • Returns coords: {'{'}latitude, longitude, accuracy{'}'}
  • watchPositionAsync() — live tracking
  • Always check status === 'granted' before calling GPS

expo-notifications

  • npx expo install expo-notifications
  • Local notifications: scheduled in-app — no server needed
  • scheduleNotificationAsync() — fire at a future time
  • Remote push: requires Expo push token + backend to send
  • Always request permission first: requestPermissionsAsync()
  • Add notification handler in App.js root to receive while app is open

Permissions Best Practices

  • Request permissions only when the user triggers the feature — not on app launch
  • Explain why before requesting: "To show your location on the map, we need GPS access"
  • Handle denial gracefully — show a fallback UI, not a crash
  • On iOS, once denied, user must go to Settings — guide them with a helpful message
import * as ImagePicker from 'expo-image-picker';
import * as Location from 'expo-location';

// Camera / gallery picker
async function pickImage() {
  const { status } = await ImagePicker.requestMediaLibraryPermissionsAsync();
  if (status !== 'granted') {
    Alert.alert('Permission required', 'Allow photo access in Settings to upload images.');
    return;
  }
  const result = await ImagePicker.launchImageLibraryAsync({
    mediaTypes: ImagePicker.MediaTypeOptions.Images,
    allowsEditing: true,
    aspect: [1, 1],
    quality: 0.8,
  });
  if (!result.canceled) setImageUri(result.assets[0].uri);
}

// Get user's current GPS location
async function getLocation() {
  const { status } = await Location.requestForegroundPermissionsAsync();
  if (status !== 'granted') { Alert.alert('GPS access denied'); return; }
  const loc = await Location.getCurrentPositionAsync({});
  setCoords(loc.coords); // { latitude, longitude, accuracy }
}

Week 7 Key Takeaways

  • Expo modules give you native hardware access without writing native code — a huge time saver
  • Always use npx expo install (not plain npm install) for Expo modules — it installs a compatible version
  • Request permissions contextually, not at app launch — users reject blanket permission requests
  • Check status === 'granted' before every native API call, every time
Week 8 — Data Persistence & Offline-First
W8

SQLite, React Query & Offline Patterns

expo-sqlite · useQuery · useMutation · optimistic updates · NetInfo

▼
Offline-first mindset: Mobile users go underground, enter lifts, and travel through dead zones constantly. An app that crashes or shows a blank screen offline is a bad app. Design as if connectivity is optional — load from local cache first, sync with the server when back online.

AsyncStorage vs SQLite

  • AsyncStorage: key-value, JSON strings, great for settings/tokens/small datasets
  • SQLite: relational database, SQL queries, great for structured data with relationships
  • Use AsyncStorage when: saving preferences, caching a small API response
  • Use SQLite when: tasks with categories, notes with tags, anything needing filtering/sorting

expo-sqlite

  • npx expo install expo-sqlite
  • openDatabaseAsync('myapp.db') — open or create DB file
  • db.execAsync('CREATE TABLE IF NOT EXISTS ...')
  • db.getAllAsync('SELECT * FROM tasks WHERE done=?', [0])
  • db.runAsync('INSERT INTO tasks (title) VALUES (?)', [text])
  • DB file persists across app restarts automatically

React Query (TanStack Query)

  • npm install @tanstack/react-query
  • useQuery({'{'}queryKey: ['posts'], queryFn: fetchPosts{'}'})
  • Returns {'{'}data, isLoading, isError, refetch{'}'}
  • Auto-caches results, re-fetches in background on focus
  • useMutation for POST/PUT/DELETE with onSuccess callbacks
  • Eliminates the boilerplate loading/error/data pattern entirely

NetInfo — Connectivity Detection

  • npm install @react-native-community/netinfo
  • NetInfo.fetch() — one-time connectivity check
  • NetInfo.addEventListener(state => ...) — subscribe to changes
  • state.isConnected — boolean
  • Show "Offline Mode" banner when disconnected
  • Queue mutations while offline, replay when reconnected
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query';
import NetInfo from '@react-native-community/netinfo';

// useQuery — fetch + cache posts automatically
const { data: posts, isLoading, isError } = useQuery({
  queryKey: ['posts'],
  queryFn: () => api.get('/posts').then(r => r.data),
  staleTime: 1000 * 60 * 5, // treat cache as fresh for 5 minutes
});

// useMutation — create a new post
const queryClient = useQueryClient();
const createPost = useMutation({
  mutationFn: (text) => api.post('/posts', { body: text }),
  onSuccess: () => queryClient.invalidateQueries({ queryKey: ['posts'] }),
});

// Offline banner
const [isOnline, setIsOnline] = useState(true);
useEffect(() => {
  return NetInfo.addEventListener(s => setIsOnline(s.isConnected));
}, []);
{!isOnline && <View style={styles.offlineBanner}><Text>Offline Mode</Text></View>}

Week 8 Key Takeaways

  • React Query removes 80% of the boilerplate around API calls — use it in any serious project
  • SQLite is a full relational database on-device — reach for it when you need queries, not just key-value
  • Monitor connectivity with NetInfo and always give users a clear signal when they are offline
  • Optimistic updates (update UI immediately, rollback if server fails) make your app feel faster
