W1

Mobile Dev Foundations

Q1.Expo Go is used to:
A Build production APKs
B Test React Native apps instantly on a physical device without building
C Submit apps to the App Store
D Write Android Java code
Expo Go scans a QR code and loads your Expo project's JavaScript bundle instantly on your device — no native build required.
Q2.In React Native, the default flexDirection is:
A row
B column
C row-reverse
D flex
Unlike CSS where the default is row, React Native defaults to column flexDirection for all Views — content stacks vertically.
Q3.Which component should be used for long scrollable lists in React Native?
A ScrollView
B ListView
C FlatList
D Map
FlatList renders only visible items (virtualization), making it memory-efficient for long lists. ScrollView renders all children at once, which is fine for short content.
Q4.StyleSheet.create() is beneficial because:
A It validates CSS syntax
B It sends styles to the native thread and enables caching
C It imports external CSS
D It auto-prefixes styles
StyleSheet.create() sends styles to the native side once, reducing bridge communication and improving performance.
Q5.Platform.OS returns:
A Device model name
B The string 'ios' or 'android'
C OS version number
D Screen density
Platform.OS returns the string 'ios' or 'android' and is used for platform-specific code branching.

W2

Core Components

Q1.Which prop handles text input changes in a controlled TextInput?
A onChange
B onInput
C onChangeText
D onValue
onChangeText receives the new string value directly; use it with useState for controlled inputs.
Q2.KeyboardAvoidingView is used to:
A Hide the keyboard on tap
B Move content up so the keyboard doesn't cover input fields
C Prevent the keyboard from opening
D Style keyboard buttons
KeyboardAvoidingView adjusts its height or padding when the keyboard appears so that inputs remain visible and accessible.
Q3.flex: 1 on a View means:
A It has a 1px border
B It takes up all available space in its parent
C It renders first
D It has 100% width
flex: 1 makes a component fill all remaining space, analogous to flex-grow: 1 in CSS.
Q4.The Image component in React Native requires:
A A CSS background-image property
B A source prop with {uri: '...'} or require('./local.png')
C An img tag instead
D Base64 encoding only
React Native's Image component accepts a source prop: use {uri} for network images and require() for local assets.
Q5.Pressable differs from TouchableOpacity by:
A It supports more gesture states and style callbacks
B It has a ripple effect by default
C It doesn't support onPress
D It only works on iOS
Pressable provides an isPressed state in its style callback, giving more control over visual feedback than TouchableOpacity's fixed opacity.

W3

Navigation

Q1.createNativeStackNavigator provides:
A A tab bar at the bottom
B A push/pop screen stack with headers
C Side drawer navigation
D Modal-only navigation
createNativeStackNavigator creates a push/pop stack navigator — screens slide in from the right, with a back button in the header.
Q2.How do you navigate to a screen named 'Profile' passing {userId: 42}?
A navigation.open('Profile', {userId:42})
B navigate.go('Profile', 42)
C navigation.navigate('Profile', {userId:42})
D router.push('/Profile/42')
navigation.navigate('ScreenName', params) is the standard React Navigation API. The params object is available as route.params in the destination.
Q3.To access params in the destination screen:
A this.props.params
B route.params
C navigation.params
D props.navigation.state.params
route.params contains the parameters passed via navigation.navigate(). Destructure with: const { userId } = route.params;
Q4.NavigationContainer must be placed:
A Inside each screen component
B As the root wrapper of the entire app
C Only in App.js screens array
D Inside SafeAreaView
NavigationContainer manages the navigation state for the whole app and must wrap all navigators — typically placed in App.js as the outermost component.
Q5.navigation.goBack() will:
A Exit the app
B Navigate to the Home screen
C Return to the previous screen in the stack
D Reload the current screen
navigation.goBack() pops the current screen off the stack and returns to the previous one — the same as pressing the hardware back button on Android.

State & Persistence

Q1.AsyncStorage stores data as:
A JSON objects directly
B Strings only — use JSON.stringify / JSON.parse
C Binary data
D Encrypted by default
Always stringify objects before storing: await AsyncStorage.setItem('key', JSON.stringify(obj)) and parse on retrieval.
Q2.useEffect with an empty dependency array [] runs:
A On every render
B Only on mount (once)
C Only on unmount
D Every second
An empty [] dependency array makes useEffect behave like componentDidMount — it runs exactly once after the component first renders.
Q3.Context API solves:
A Performance bottlenecks
B Prop drilling — passing state through many component levels
C AsyncStorage limitations
D Network requests
Context allows you to share state globally without passing props down every level. Auth state, theme, and language preferences are common use cases.
Q4.useReducer is preferred over useState when:
A State is a single boolean
B State is a string
C State has multiple sub-values with complex transitions
D You don't need re-renders
useReducer is ideal when state updates depend on the previous state or when multiple related values update together — think cart, form with many fields, or todo list.
Q5.To load AsyncStorage data when the component mounts:
A useLayoutEffect(() => loadData())
B useEffect(() => { loadData(); }, [])
C componentDidMount()
D onMount={loadData}
useEffect with an empty array is the hooks equivalent of componentDidMount. Wrap async calls in an inner async IIFE or separate function.

W5

APIs & Networking

Q1.fetch() in React Native is:
A A third-party library
B The same Web Fetch API, built into React Native
C Only available on iOS
D A polyfill you must install
React Native ships with the standard Fetch API built in. You can use it the same way as in a browser — no installation needed.
Q2.To send a JSON POST request with fetch:
A fetch(url, {type:'POST', data:body})
B fetch(url, {method:'POST', headers:{'Content-Type':'application/json'}, body:JSON.stringify(data)})
C fetch.post(url, data)
D axios.post is required instead
You must set method:'POST', the Content-Type header, and JSON.stringify the body. Axios simplifies this but fetch works the same way.
Q3.ActivityIndicator is used for:
A Tracking user activity
B Displaying a loading spinner
C Measuring app performance
D Checking network speed
ActivityIndicator renders the platform's native loading spinner. Use it with a loading boolean state while waiting for API responses.
Q4.AbortController is used to:
A Stop the app
B Cancel a pending fetch request (typically on component unmount)
C Block network access
D Force re-render
AbortController lets you cancel a fetch that's still in-flight. Return () => controller.abort() from useEffect to clean up on unmount.
Q5.React Query's useQuery returns:
A Just the data
B isLoading, isError, data, refetch, and more
C A Promise
D A callback
useQuery returns a rich object with status flags (isLoading, isError, isFetching), the data, an error object, and a refetch function.
