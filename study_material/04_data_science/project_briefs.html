P1

Exploratory Data Analysis Report

Beginner Due: End of Week 4 100 Points Pass: 60 pts

Overview

Produce a professional exploratory data analysis (EDA) report as a Jupyter Notebook. This project tests your ability to load real data, clean it systematically, ask meaningful questions about it, answer those questions with visualizations, and communicate your findings clearly in writing.

Tech Stack

Python 3.x Jupyter Notebook / Colab pandas Matplotlib Seaborn GitHub

Dataset Options

Choose ONE of the following (all available without external downloads):

  • Titanic — sns.load_dataset('titanic') — Passenger survival data. Rich mix of numeric, categorical, and missing values.
  • Iris — from sklearn.datasets import load_iris — Classic flower classification dataset. Clean and well-structured.
  • California Housing — from sklearn.datasets import fetch_california_housing — House price prediction with 8 numeric features.

Requirements Checklist

  • ✓ Load the dataset and display shape, dtypes, and head()
  • ✓ Handle missing values (explain your strategy for each column)
  • ✓ Remove duplicates and fix inconsistent data types
  • ✓ At least 8 meaningful visualizations (minimum: 1 histogram, 1 scatter, 1 heatmap, 1 barplot/boxplot)
  • ✓ Written Markdown analysis cell below each chart (what does it show? what is your key observation?)
  • ✓ A "5 Key Insights" Markdown section at the end with numbered bullet points
  • ✓ Exported HTML report: jupyter nbconvert --to html
  • ✓ GitHub repository with a README that describes the project, dataset, and top 3 findings

Grading Rubric

CategoryPointsWhat We're Looking For
Data Cleaning 20 Missing values handled with reasoning (not just dropped), duplicates removed, types fixed. Clean df shown at end.
Visualizations 25 8+ charts, correct chart type for each question, labeled axes and titles, readable colors, charts saved as PNG.
Written Analysis 25 Each chart has a Markdown cell with a genuine observation — not just "this chart shows X." 5 Key Insights are specific, not generic.
Code Quality 15 Clean, readable code. Functions used to avoid repetition. No dead/commented-out code. Comments where needed.
Presentation 15 Notebook has a title cell, logical section flow, all cells executed in order with output visible, HTML export clean.
Total100Pass threshold: 60 pts
Submission: GitHub repository URL submitted via intern portal. Deadline: Sunday midnight of Week 4. Late submissions: -5 pts per day.
P2

Predictive Model — Customer Churn

Intermediate Due: End of Week 8 100 Points Pass: 65 pts

Overview

Build a complete machine learning pipeline to predict customer churn for a telecom company. This project tests your end-to-end ML skills: data preparation, feature engineering, model training, rigorous evaluation, and interpretation. The deliverable is a production-ready prediction system that a business team could actually use.

Tech Stack

Python 3.x Jupyter Notebook pandas / NumPy scikit-learn XGBoost Matplotlib / Seaborn pickle GitHub

Dataset

Telco Customer Churn — Available on Kaggle (search "Telco Customer Churn" by IBM). Your mentor will share the CSV file via the intern portal if you cannot access Kaggle. The dataset contains ~7,000 customers with 20+ features including demographics, account information, and services subscribed.

Key columns: customerID, tenure, MonthlyCharges, TotalCharges, Contract, InternetService, Churn (target — Yes/No).

Requirements Checklist

  • ✓ Full EDA section: distribution of churn, class balance, correlation with key features
  • ✓ Feature engineering: encode all categoricals, handle TotalCharges (stored as string), create at least 2 new features
  • ✓ Handle class imbalance (churn ~26%) — use class_weight='balanced' or SMOTE
  • ✓ Train at least 3 models: Logistic Regression, Random Forest, XGBoost
  • ✓ Compare using: Accuracy, Precision, Recall, F1, ROC-AUC in a summary table
  • ✓ Tune the best model with GridSearchCV or RandomizedSearchCV (show before/after performance)
  • ✓ Final model saved as churn_model.pkl using pickle
  • ✓ A predict_churn(customer_data) function that accepts a dict and returns churn probability
  • ✓ GitHub repo with README, requirements.txt, and notebook

Grading Rubric

CategoryPointsWhat We're Looking For
EDA Quality 20 Churn rate analysis, correlation of churn with each feature, visualizations for key business questions, written observations.
Feature Engineering 20 All categoricals encoded correctly, TotalCharges converted to float, at least 2 new features with justification, class imbalance handled.
Model Comparison 25 3+ models trained, compared on 5 metrics in a table, ROC curves plotted together, reasoning for metric choice given the business context.
Tuning & Final Model 20 GridSearchCV with reasonable parameter grid, clear before/after comparison, model saved as pickle, prediction function works on new customer dict.
Code Quality 15 Modular code (functions, not copy-paste), clear variable names, no unused imports, informative print statements, GitHub README complete.
Total100Pass threshold: 65 pts

Mentor Tips

  • F1-score matters more than accuracy here — churn is imbalanced. A model that always predicts "No Churn" gets 74% accuracy but catches zero churners.
  • Think about the business cost: missing a churning customer (false negative) is worse than flagging a non-churner (false positive). Adjust your threshold accordingly.
  • In your EDA, check: do contract type and tenure interact with churn? These tend to be the strongest predictors.
Submission: GitHub repository URL submitted via intern portal. Deadline: Sunday midnight of Week 8. Include a requirements.txt so mentors can reproduce your environment.
P3

End-to-End Data Product

Advanced Due: End of Week 12 100 Points Pass: 70 pts

Overview

Build a complete, production-deployed data product: an ML prediction API backed by a Flask server, an interactive Dash or Streamlit dashboard with 5+ charts and a live prediction form, all containerized with Docker and deployed live to a cloud hosting platform. This project proves you can ship real data science work, not just notebooks.

Tech Stack

Python 3.x scikit-learn / XGBoost Flask (API) Dash or Streamlit (Dashboard) SQLite / PostgreSQL Docker GitHub Actions (CI) Heroku / Render / Railway (Deploy)

Dataset

Any real public dataset of your choice. Recommended domains (choose what interests you):

  • Heart Disease — Medical prediction, widely understood
  • Loan Default Prediction — Financial risk, very relevant for India
  • Air Quality Index — Environmental data, relevant for Devbhoomi region
  • Crop Yield Prediction — Agriculture, highly relevant to Uttarakhand context
  • E-commerce Sales Forecasting — Combine time series + dashboard

Requirements Checklist

  • ✓ Clean, modeled dataset with feature engineering and cross-validated ML model (AUC > 0.75 or R² > 0.70)
  • ✓ Flask REST API: POST /predict endpoint accepts JSON, returns prediction + probability + timestamp
  • ✓ SQLite or PostgreSQL: log every prediction (input features, prediction, timestamp) to a database
  • ✓ Dashboard (Dash or Streamlit): 5+ charts including at least one interactive filter/dropdown, and a prediction form where user enters values and sees model output live
  • ✓ Docker: Dockerfile that builds and runs the entire app on any machine
  • ✓ GitHub Actions: CI workflow that runs at least one test (pytest) on every push to main
  • ✓ Deployed live: share a public URL where mentors can test your app without running it locally
  • ✓ 1-page business report (PDF or Markdown): problem, approach, key findings, 3 business recommendations

Flask API Example Structure

>from flask import Flask, request, jsonify
import pickle, sqlite3, datetime
import pandas as pd

app = Flask(__name__)

with open('model.pkl', 'rb') as f:
    model = pickle.load(f)

def log_prediction(features, prediction, probability):
    conn = sqlite3.connect('predictions.db')
    conn.execute('''CREATE TABLE IF NOT EXISTS predictions
                    (id INTEGER PRIMARY KEY, timestamp TEXT, features TEXT,
                     prediction INTEGER, probability REAL)''')
    conn.execute('INSERT INTO predictions VALUES (?,?,?,?,?)',
                 (None, datetime.datetime.now().isoformat(),
                  str(features), int(prediction), float(probability)))
    conn.commit()
    conn.close()

@app.route('/predict', methods=['POST'])
def predict():
    data = request.get_json()
    df = pd.DataFrame([data])
    prediction   = int(model.predict(df)[0])
    probability  = float(model.predict_proba(df)[0][1])
    log_prediction(data, prediction, probability)
    return jsonify({
        'prediction':  prediction,
        'probability': round(probability, 3),
        'timestamp':   datetime.datetime.now().isoformat()
    })

@app.route('/health', methods=['GET'])
def health():
    return jsonify({'status': 'ok', 'model': 'loaded'})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

Mentor Tips

  • Start with a working notebook, then extract the model into a Flask API. Don't try to build the API and model simultaneously.
  • Render.com and Railway.app both have free tiers that support Docker deployments — much easier than Heroku's current setup.
  • Your business report should be written for a CEO, not a data scientist. No model equations — just findings and recommendations.
  • If CI/CD is new to you: the GitHub Actions workflow just needs to pip install and run one pytest function that calls your /predict endpoint and checks the response format.
