The Artificial Neuron

A single neuron computes: output = activation( w₁x₁ + w₂x₂ + ... + wₙxₙ + b )

Stacking neurons in layers creates a network that can approximate any continuous function (Universal Approximation Theorem). Depth enables hierarchical feature learning: raw pixels → edges → shapes → object parts → objects.

Activation Functions

  • ReLU: f(x) = max(0, x). Default for hidden layers. Computationally trivial, no vanishing gradient for positive x, produces sparse activations. Use in all hidden layers of feedforward and convolutional networks.
  • Leaky ReLU: f(x) = max(0.01x, x). Fixes "dying ReLU" — neurons that produce 0 for all inputs and never recover during training (all gradients are 0).
  • Sigmoid: σ(x) = 1/(1+e⁻ˣ). Output 0–1. Used in LSTM/GRU gates, binary classification output layer. Suffers severe vanishing gradients in deep networks.
  • Tanh: f(x) = (eˣ−e⁻ˣ)/(eˣ+e⁻ˣ). Output −1 to 1. Zero-centered (better gradient flow than sigmoid). Used in RNN hidden states.
  • Softmax: f(xᵢ) = eˣⁱ/Σeˣʲ. Converts raw logits to probability distribution summing to 1. Always used in multiclass output layer.
  • GELU: Gaussian Error Linear Unit — smooth approximation of ReLU used in BERT, GPT, and all modern Transformers. Slightly better empirical performance on NLP tasks.

Backpropagation — Chain Rule

Backprop computes ∂Loss/∂w for every weight using the chain rule, propagating gradients from output to input. Then gradient descent updates: w = w − η × ∂Loss/∂w

>import numpy as np

# 2-layer neural net forward + backward pass (educational)
def relu(x): return np.maximum(0, x)
def relu_d(x): return (x > 0).astype(float)
def sigmoid(x): return 1 / (1 + np.exp(-x))
def sigmoid_d(x): s=sigmoid(x); return s*(1-s)

np.random.seed(42)
X = np.array([[0.5, 0.3]])   # 1 sample, 2 features
y = np.array([[1.0]])

W1 = np.random.randn(2, 4) * 0.1   # 2 inputs → 4 hidden
b1 = np.zeros((1, 4))
W2 = np.random.randn(4, 1) * 0.1   # 4 hidden → 1 output
b2 = np.zeros((1, 1))
lr = 0.01

# Forward pass
z1 = X @ W1 + b1       # (1,4) pre-activation
a1 = relu(z1)          # (1,4) hidden layer output
z2 = a1 @ W2 + b2      # (1,1) output pre-activation
a2 = sigmoid(z2)       # (1,1) final probability

# Binary Cross-Entropy Loss
loss = -(y * np.log(a2 + 1e-9) + (1-y) * np.log(1-a2 + 1e-9))
print(f"Loss before: {loss.item():.4f}")

# Backward pass (chain rule)
dL_da2 = -(y / a2) + (1-y) / (1-a2)
dL_dz2 = dL_da2 * sigmoid_d(z2)
dL_dW2 = a1.T @ dL_dz2
dL_db2 = dL_dz2
dL_da1 = dL_dz2 @ W2.T
dL_dz1 = dL_da1 * relu_d(z1)
dL_dW1 = X.T @ dL_dz1
dL_db1 = dL_dz1

# Update weights
W1 -= lr * dL_dW1; b1 -= lr * dL_db1
W2 -= lr * dL_dW2; b2 -= lr * dL_db2
print("Backward pass complete — weights updated.")

Optimizers — From SGD to Adam

  • SGD: Update with one random sample's gradient. Noisy but each step is fast. lr=0.01 typical. Converges slowly.
  • SGD + Momentum: Exponential moving average of gradients. Builds speed in consistent directions, dampens oscillations. momentum=0.9 standard.
  • RMSprop: Divides gradient by running RMS of squared gradients (per-parameter adaptive LR). Handles non-stationary objectives. Good for RNNs.
  • Adam: Combines Momentum (first moment m̂) + RMSprop (second moment v̂). Default optimizer for almost all deep learning. β₁=0.9, β₂=0.999, ε=1e-8.
  • AdamW: Adam + decoupled weight decay (L2 regularization applied separately, not through gradient). Preferred for Transformers. PyTorch's AdamW.

Regularization in Deep Learning

>import tensorflow as tf
from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([
    layers.Dense(256, input_shape=(100,)),

    # Batch Normalization: normalize layer inputs (mean≈0, std≈1 per batch)
    # Benefits: faster training, higher learning rate, acts as regularization
    # Place BEFORE activation (original paper) or after (common practice varies)
    layers.BatchNormalization(),
    layers.Activation('relu'),

    # Dropout: randomly zeros p% of neurons during training
    # Forces network to learn redundant representations → generalizes better
    # rate=0.2–0.5 for Dense; 0.1–0.3 for Conv
    layers.Dropout(0.3),

    layers.Dense(128, activation='relu',
                 kernel_regularizer=keras.regularizers.l2(1e-4)),  # L2 weight decay
    layers.BatchNormalization(),
    layers.Dropout(0.2),

    layers.Dense(10, activation='softmax')
])

model.compile(
    optimizer=keras.optimizers.Adam(learning_rate=1e-3),
    loss='sparse_categorical_crossentropy',
    metrics=['accuracy']
)

history = model.fit(
    X_train, y_train, epochs=50, batch_size=32, validation_split=0.15,
    callbacks=[
        keras.callbacks.EarlyStopping(patience=5, restore_best_weights=True),
        keras.callbacks.ReduceLROnPlateau(factor=0.5, patience=3, verbose=1),
        keras.callbacks.ModelCheckpoint('best_model.keras', save_best_only=True)
    ]
)
print(f"Best val accuracy: {max(history.history['val_accuracy']):.3f}")

▼

Why CNNs for Images?

A 224×224×3 image = 150,528 values. A Dense layer with 1024 neurons needs 154M parameters — expensive and overfitting-prone. CNNs solve this with three key ideas:

  • Local Feature Detection: 3×3 or 5×5 filters look at small regions at a time. Early layers learn edges and gradients, middle layers learn shapes and textures, deep layers learn semantic parts (eyes, wheels, fins).
  • Weight Sharing: The same 3×3 filter (9 weights) slides across the entire image. A 32-filter Conv2D layer needs only 32×9 = 288 parameters regardless of image size.
  • Translation Invariance: A cat filter activates whether the cat is top-left or bottom-right — same learned features apply everywhere.

CNN Architecture

>import tensorflow as tf
from tensorflow.keras import layers, models

# CNN for CIFAR-10 (32×32 RGB images, 10 classes)
model = models.Sequential([
    # Conv output size: (H − k + 2p)/stride + 1; padding='same' keeps H,W constant
    layers.Conv2D(32, (3,3), activation='relu', padding='same', input_shape=(32,32,3)),
    layers.BatchNormalization(),
    layers.Conv2D(32, (3,3), activation='relu', padding='same'),
    layers.MaxPooling2D((2,2)),   # 32×32 → 16×16 (halves spatial dims)
    layers.Dropout(0.2),

    layers.Conv2D(64, (3,3), activation='relu', padding='same'),
    layers.BatchNormalization(),
    layers.Conv2D(64, (3,3), activation='relu', padding='same'),
    layers.MaxPooling2D((2,2)),   # 16×16 → 8×8
    layers.Dropout(0.3),

    layers.Conv2D(128, (3,3), activation='relu', padding='same'),
    layers.BatchNormalization(),
    layers.MaxPooling2D((2,2)),   # 8×8 → 4×4
    layers.Dropout(0.4),

    layers.Flatten(),             # 4×4×128 = 2048 values → flat vector
    layers.Dense(256, activation='relu'),
    layers.BatchNormalization(),
    layers.Dropout(0.5),
    layers.Dense(10, activation='softmax')  # 10 CIFAR classes
])
model.summary()

Transfer Learning

Pre-trained models (ImageNet: 1.2M images, 1000 classes) have learned universal visual features. Fine-tuning repurposes them for your task in hours instead of weeks from scratch.

model.compile(optimizer=optimizers.Adam(1e-3),

Week 11 — Transformers, LLMs & Generative AI

▼

The Attention Mechanism

Attention answers: "For understanding this token, which other tokens are most relevant?" This replaced the RNN bottleneck of compressing entire sequences into a single fixed-size vector.

Scaled Dot-Product Attention:

Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V

  • Q (Query): "What am I searching for?" — current token's question vector
  • K (Key): "What can I offer?" — each position's searchable tag
  • V (Value): "What is my actual content?" — information to aggregate
  • √d_k scaling: Prevents very large dot products → unstable softmax → vanishing gradients
  • Multi-Head Attention: Run attention h times in parallel with different learned projections. h=8 in original Transformer, h=16–32 in large models. Captures different relationship types simultaneously.

BERT vs GPT Architecture

  • BERT (Google, 2018): Bidirectional Encoder — reads the entire sequence simultaneously. Pre-trained with Masked Language Modeling (predict randomly masked tokens). Best for: text classification, NER, QA (extractive), semantic similarity, embeddings.
  • GPT (OpenAI, 2018–2024): Autoregressive Decoder — generates tokens one at a time, left-to-right. Pre-trained to predict the next token. Best for: text generation, summarization, translation, code, chat, instruction following.

RAG — Retrieval-Augmented Generation

LLMs have a training data cutoff and can hallucinate facts. RAG grounds responses in real, up-to-date documents by retrieving relevant text before generating.

># RAG System Pseudocode (conceptual)

# --- INDEXING PHASE (run once) ---
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.embeddings import HuggingFaceEmbeddings
import chromadb

# 1. Load and split documents into ~500-token chunks
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(documents)

# 2. Embed chunks as vectors
embedder = HuggingFaceEmbeddings(model_name='all-MiniLM-L6-v2')

# 3. Store in vector database
client = chromadb.PersistentClient(path="./rag_db")
collection = client.get_or_create_collection("knowledge_base")
for i, chunk in enumerate(chunks):
    vec = embedder.embed_query(chunk.page_content)
    collection.add(ids=[str(i)], embeddings=[vec], documents=[chunk.page_content])

# --- QUERY PHASE (each user request) ---
import anthropic

def rag_answer(user_query: str) -> str:
    # 4. Embed user question
    query_vec = embedder.embed_query(user_query)

    # 5. Retrieve top-5 most relevant chunks
    results = collection.query(query_embeddings=[query_vec], n_results=5)
    context = "\n\n".join(results['documents'][0])

    # 6. Augment prompt with retrieved context
    prompt = f"""Answer based ONLY on the following context.
If the answer is not in the context, say "I don't have information on that."

Context:
{context}

Question: {user_query}"""

    # 7. Generate grounded answer
    client = anthropic.Anthropic()
    msg = client.messages.create(
        model="claude-3-5-sonnet-20241022",
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}]
    )
    return msg.content[0].text

The Notebook-to-Production Gap

Only 22% of ML models ever reach production (Gartner, 2022). Key challenges: reproducibility (which data version? which library versions?), scalability (1 request in Colab vs 10,000/sec in production), monitoring (model silently degrades), governance (who approved this model?). MLOps addresses all of these systematically.

FastAPI Model Serving

># app.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, validator
import joblib, numpy as np, logging

logger = logging.getLogger(__name__)
app = FastAPI(title="Churn Prediction API", version="2.0.0")

model = joblib.load("churn_pipeline.pkl")  # Load at startup

class ChurnRequest(BaseModel):
    tenure_months:    int
    monthly_charges:  float
    total_charges:    float
    num_products:     int
    has_internet:     bool

    @validator('monthly_charges', 'total_charges')
    def must_be_positive(cls, v):
        if v <= 0: raise ValueError("Charges must be positive")
        return v

class ChurnResponse(BaseModel):
    churn_probability: float
    prediction:        str
    risk_tier:         str

@app.post("/predict", response_model=ChurnResponse)
async def predict(req: ChurnRequest):
    try:
        features = [[req.tenure_months, req.monthly_charges,
                     req.total_charges, req.num_products, int(req.has_internet)]]
        prob = float(model.predict_proba(features)[0][1])
        pred = "CHURN" if prob > 0.5 else "RETAIN"
        tier = "HIGH" if prob > 0.7 else "MEDIUM" if prob > 0.4 else "LOW"
        logger.info(f"predict | {pred} | {prob:.3f}")
        return ChurnResponse(churn_probability=round(prob,4),
                             prediction=pred, risk_tier=tier)
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

@app.get("/health")
async def health(): return {"status": "healthy"}

# Run: uvicorn app:app --host 0.0.0.0 --port 8000 --workers 4
