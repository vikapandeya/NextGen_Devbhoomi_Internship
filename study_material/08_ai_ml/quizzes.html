Week 1 — AI Fundamentals
5 questions · AI types, ethics, timeline

Q1. Which of the following BEST describes the relationship between AI, Machine Learning, and Deep Learning?

Correct! The hierarchy is: AI (broadest) → Machine Learning (subset of AI) → Deep Learning (subset of ML using neural networks). Every DL system is an ML system is an AI system, but not vice versa.

Q2. A bank's credit scoring system assigns loan approval probabilities. What type of machine learning is this?

Correct! Credit scoring is supervised learning — the model is trained on historical loan data where each application has a known outcome (defaulted or repaid). The labeled data directly teaches the model which patterns predict default.

Q3. India's Digital Personal Data Protection (DPDP) Act 2023 requires organizations to do which of the following?

Correct! DPDP Act 2023 mandates: explicit consent before collection, purpose limitation (use only for stated purpose), right to erasure, data breach notification within 72 hours, and fines up to ₹250 crore for violations.

Q4. Which event in 2012 is widely considered the beginning of the modern Deep Learning revolution?

Correct! In 2012, AlexNet (Geoffrey Hinton's team at Toronto) achieved 15.3% error on ImageNet vs. 26% for the second-place entry — a 41% improvement. This demonstrated that deep CNNs trained on GPUs were dramatically superior, triggering massive investment in deep learning research.

Q5. In the context of AI ethics, what is "algorithmic bias"?

Correct! Algorithmic bias occurs when an AI system produces systematically prejudiced results — often because training data reflects historical inequalities. Example: Amazon's hiring AI penalized resumes mentioning "women's" organizations because historical hiring data was male-dominated.
W2
Week 2 — Python & ML Workflow
5 questions · sklearn, train-test split, ML pipeline

Q1. What is the purpose of setting stratify=y in train_test_split()?

Correct! stratify=y preserves the class distribution from the full dataset in both train and test splits. Without it, on an imbalanced dataset (e.g., 95% class 0, 5% class 1), the test set might randomly contain no class 1 examples, giving a misleading 100% accuracy.

Q2. In sklearn's ML API, what do fit() and transform() do respectively for a StandardScaler?

Correct! fit() computes statistics from the training data (mean and std for StandardScaler). transform() applies those learned statistics to scale the data. On test data, you only call transform() (never fit()) to avoid data leakage.

Q3. What shape should the feature matrix X be when passed to a sklearn model?

Correct! sklearn expects X to be a 2D array (or DataFrame) of shape (n_samples, n_features) — rows are samples, columns are features. The target y is 1D with shape (n_samples,). A common error is accidentally passing a 1D array, which causes a ValueError.

Q4. In Google Colab, how do you enable GPU acceleration for faster model training?

Correct! In Google Colab, GPU must be explicitly enabled via Runtime menu → Change runtime type → Hardware accelerator → T4 GPU → Save. Verify with !nvidia-smi to see the GPU details. The free tier provides a Tesla T4 with ~15GB VRAM.

Q5. Which step in the ML workflow is MOST important for preventing a model from "looking good in testing but failing in production"?

Correct! Proper train-test separation (no data leakage) is the most critical step. Data leakage causes models to appear much better in testing than they perform in production, because they've indirectly "seen" the test data during training. Use sklearn Pipeline to enforce this.
W3
Week 3 — Data Preprocessing
5 questions · missing values, encoding, scaling

Q1. A dataset column "City" has 500 unique values. You need to encode it for a Logistic Regression model. What is the BEST approach?

Correct! For high-cardinality categoricals (many unique values), target encoding is preferred. LabelEncoder implies a false ordinal relationship (Mumbai=0 < Delhi=1). OneHotEncoding 500 cities creates 500 sparse, mostly-useless columns. Target encoding captures the categorical's relationship with the target while staying compact.

Q2. When should you use MinMaxScaler instead of StandardScaler?

Correct! MinMaxScaler maps all values to [0, 1] and is preferred for neural networks with sigmoid/tanh activations that expect bounded inputs, or image pixel normalization (0–255 → 0–1). However, it's highly sensitive to outliers — one extreme value compresses all other values into a tiny range. StandardScaler is more robust for most ML use cases.

Q3. Your dataset has 30% of values missing in the "income" column. What is the BEST strategy?

Correct! The best approach: impute missing values with median (robust to income outliers unlike mean), AND add a binary indicator column "income_was_missing" = 1. This preserves information about the missingness itself — the pattern of who has missing income may be predictive.

Q4. Why does sklearn's Pipeline prevent data leakage in cross-validation?

Correct! When cross_val_score runs a Pipeline, in each fold the Pipeline's fit_transform is called on the training fold only, and transform (no fit) is called on the validation fold. This mirrors production deployment and prevents validation data statistics from contaminating the preprocessing step.

Q5. A dataset for fraud detection has 99% non-fraud and 1% fraud. A model that always predicts "no fraud" achieves 99% accuracy. What metric should you use instead?

Correct! On imbalanced datasets, accuracy is misleading. A model predicting "no fraud" always gets 99% accuracy but catches 0% of actual fraud. F1 score and PR-AUC both heavily penalize this. PR-AUC is particularly good because it focuses on the rare positive class performance rather than the majority class.
W4
Week 4 — First ML Models
5 questions · logistic regression, metrics, bias-variance

Q1. A medical test for a rare disease has 99% sensitivity (recall) but only 10% precision. What does this mean in practice?

Correct! 99% recall (sensitivity) = catches 99% of true sick patients (very few false negatives). 10% precision = of all patients flagged positive, only 10% are actually sick (90% are healthy false alarms). This happens with rare diseases — even a fairly accurate test overwhelms true positives with false alarms on a mostly-healthy population.

Q2. What does the R² score of 0.75 mean for a regression model?

Correct! R² (coefficient of determination) measures the proportion of variance in the target variable explained by the model. R²=0.75 means the model accounts for 75% of the variance — better than simply predicting the mean (R²=0). R²=1.0 is perfect. R²<0 means the model is worse than predicting the mean.

Q3. A model has 100% training accuracy but only 60% test accuracy on the same problem. This is a case of:

Correct! This is classic overfitting (high variance). The large gap between 100% training and 60% test accuracy indicates the model memorized training data including noise and outliers, but failed to learn the underlying patterns. Fix: regularization (dropout, L2 penalty), simpler model, more training data, or cross-validation based hyperparameter tuning.

Q4. The sigmoid function outputs a value of 0.85 for a logistic regression model predicting loan default. With a threshold of 0.5, what does the model predict?

Correct! With threshold=0.5, the model predicts class 1 (Default) for any probability above 0.5. Since 0.85 > 0.5, the prediction is Default. The model also conveys its confidence: 85% probability of default is quite high. The threshold can be adjusted — a bank might use 0.3 to be more conservative (catch more potential defaults at the cost of more false alarms).

Q5. Which of the following is a correct way to fix underfitting in a neural network?

Correct! Underfitting means the model lacks capacity to learn the patterns (high bias). The fix is to increase model complexity: add more layers, add more neurons, use a more powerful architecture, remove excessive regularization, or train for more epochs. Note: Dropout increases regularization (which helps overfitting, not underfitting). More data helps when the model has sufficient capacity but insufficient examples.
W5
Week 5 — Ensemble Methods
5 questions · Random Forest, XGBoost, SHAP

Q1. What is the key difference between Random Forest (bagging) and Gradient Boosting?

Correct! This is the fundamental distinction: Random Forest builds N trees independently in parallel (bagging) and averages their predictions — reducing variance. Gradient Boosting builds trees sequentially, with each new tree focusing on the residual errors of the current ensemble — reducing bias. Both use decision trees but with opposite ensemble strategies.

Q2. In XGBoost, what does early_stopping_rounds=20 do?

Correct! early_stopping_rounds monitors validation metric (e.g., AUC) after each boosting round. If the metric doesn't improve for 20 consecutive rounds, training stops and the model with the best validation score is restored. This prevents overfitting without manual n_estimators tuning — a critical technique for gradient boosting in production.

Q3. A feature with a high positive SHAP value for a specific prediction means:

Correct! SHAP values explain individual predictions by showing each feature's contribution relative to the model's base rate. A high positive SHAP value for a feature means "this feature's value pushed the prediction ABOVE the average prediction for this specific sample." SHAP is local (per-prediction), not global (unlike feature_importances_ which describes overall model behavior).

Q4. When training a Random Forest, what does the "out-of-bag" (OOB) score estimate?

Correct! Bootstrap sampling means each tree only sees ~63.2% of training data. The remaining ~36.8% are "out-of-bag" for that tree. By averaging each sample's predictions across all trees it was OOB for, Random Forest gets a nearly free cross-validation estimate of generalization performance without needing a separate validation set. Set oob_score=True in sklearn.

Q5. Why does LightGBM often train faster than XGBoost on large datasets?

Correct! XGBoost uses level-wise growth (splits all leaves at the same depth simultaneously). LightGBM uses leaf-wise growth — it finds and splits the single leaf that reduces loss the most, regardless of level. This achieves lower loss with fewer splits. LightGBM also uses histogram-based splits (GOSS gradient-based sampling) that dramatically reduce memory and computation on large datasets.
