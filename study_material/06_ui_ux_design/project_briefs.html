Project Briefs

Three graded projects — one per level — taking you from UX audit to production-ready design handoff. Each project builds on the previous.

Beginner

P1 — App Audit + Redesign Proposal

Deadline: Week 4 Pass Mark: 60 / 100
Tools Required: Figma (free) Google Forms Screen recording app

Description

Choose any poorly designed Indian app or government portal — IRCTC, a state transport website, a local municipality portal, or a regional government service. Conduct a thorough UX audit using established heuristics and propose improvements with wireframes. This project teaches you to see UX problems that users feel but cannot name.

Requirements

  • Conduct a full heuristic evaluation against Nielsen's 10 Usability Heuristics. Document every finding with a severity rating (1 = cosmetic, 2 = minor, 3 = major, 4 = catastrophic) and a screenshot annotated with what the problem is.
  • Create 3 user interview questions and conduct 3 interviews with classmates. Written notes are acceptable — you do not need a video recording. Sample questions: "Walk me through the last time you used this app. Where did you get stuck?" "What were you trying to do? Did you succeed?"
  • Identify 5 specific UX problems from your combined heuristic evaluation and interview findings. For each problem: provide an annotated screenshot, name the violated heuristic, and rate severity.
  • Write a 1-paragraph problem statement using this structure: "Users of [product] who are trying to [task] struggle with [specific problem] because [root cause]. This results in [negative outcome]."
  • Create lo-fi wireframes in Figma for 3 redesigned screens that address your top 3 identified problems. Greyscale, no colour needed at this stage.
  • Write a 200-word rationale explaining your design decisions: why did you change each element? What heuristic does your redesign now satisfy?

Grading Rubric

CriterionMarksWhat Examiners Look For
Heuristic Evaluation25All 10 heuristics checked against the chosen app; issues documented with severity ratings and annotated screenshots
User Research253 interviews conducted with relevant questions; findings summarised showing patterns across participants
Problem Identification255 distinct UX problems identified with supporting evidence (screenshot + heuristic + severity); problem statement clearly written
Redesign Wireframes253 Figma wireframes addressing identified problems; design decisions explained in rationale
Total100Pass mark: 60. Strong work clearly documents the connection between research findings and design decisions.
Intermediate

P2 — Complete Mobile App Design

Deadline: Week 8 Pass Mark: 65 / 100
Tools Required: Figma Stark plugin (accessibility checker)

Description

Design a complete mobile app from scratch — your choice of concept. The only constraint is that it must solve a real problem for a real user group. Deliver a full design package from persona to interactive prototype. This is your first portfolio-quality piece.

Suggested concepts: A local government service (renewing a driving licence), a college canteen ordering app, a neighbourhood marketplace, a farmer's produce price tracker, a local bus route planner.

Requirements

  • 1 User Persona — name, photo placeholder, age, occupation, goals (2–3), frustrations (2–3), a direct quote, tech comfort level. Make it believable — base it on a real person you interviewed or observed.
  • 1 User Journey Map — stages of the task, user actions at each stage, thoughts, emotions (use an emotional curve), pain points, and design opportunities.
  • Sitemap — hierarchical map of all screens in your app, showing navigation structure and relationships between sections.
  • Lo-fi wireframes for all screens — greyscale, annotated, every screen in the core user flow plus key secondary screens.
  • Hi-fi mockups for all screens — full colour, real content (no Lorem Ipsum), consistent with your mini design system.
  • Interactive Figma prototype — must cover the complete core user flow end-to-end, clickable with smooth transitions. A reviewer must be able to complete the primary task without any guidance from you.
  • Mini design system page — colour palette (named swatches), typography scale, and at least 5 reusable components with documented states.
  • Accessibility audit using Stark plugin — run Stark on every screen. All text must pass WCAG AA contrast ratio (4.5:1 for body, 3:1 for large text). Document findings and fixes.
  • 5 usability test observations — test your prototype with classmates (5 participants). For each: note what task you gave them, whether they completed it, and one key observation.

Grading Rubric

CriterionMarksWhat Examiners Look For
Research Artifacts15Persona grounded in real observations, not stereotypes; journey map shows emotional arc and opportunities
Information Architecture15Sitemap covers all screens with logical hierarchy; navigation labels match user language
Visual Design Quality25Consistent design system applied throughout; spacing, typography, and colour used intentionally; real content throughout
Prototype20Core flow is fully clickable without guidance; Smart Animate or appropriate transitions used; feels like a real app
Accessibility15Stark audit shows AA pass on all screens; findings documented; issues fixed with evidence
Process Documentation10Figma pages organised logically; design decisions explained in annotations or a brief write-up
Total100Pass mark: 65. Outstanding projects show clear connections between research, design decisions, and test feedback.
Advanced

P3 — End-to-End Design Handoff Package

Deadline: Week 12 Pass Mark: 70 / 100
Tools Required: Figma Tokens Studio plugin Stark plugin Behance or Notion

Description

Design a complete SaaS dashboard OR e-commerce mobile app. Deliver a production-ready design package suitable for a development team to implement without needing to ask you clarifying questions. This is your capstone project and primary portfolio piece.

SaaS Dashboard options: HR management, inventory tracking, analytics platform, project management, support ticket system.
E-commerce options: Vertical marketplace (books, handloom, organic produce), service booking app, B2B procurement.

Requirements

  • Full research phase: Use at least 2 methods (e.g. user interviews + competitive analysis). Produce a synthesis document showing key insights that shaped your design direction.
  • Complete design system in Figma: Colour tokens (global + semantic), typography scale, spacing system, 8+ components with full variant sets, anatomy documentation, and usage guidelines for each component.
  • Hi-fi interactive prototype: All major flows must be fully clickable (core flow + at least 1 secondary flow such as error state or empty state).
  • Developer handoff spec: Annotated screens showing spacing grid, component labels with props, asset export list (SVG icons + image assets named correctly), and a component inventory table.
  • UX writing guide: Document your product's voice and tone (3 adjectives + 3 anti-adjectives). Document 20 UI strings with before/after examples showing the correct tone applied.
  • WCAG 2.1 AA accessibility report: Stark audit on all screens. All contrast failures fixed. Keyboard navigation flow documented. Report must include: number of issues found, severity breakdown, and evidence of all issues resolved.
  • Moderated usability test with 3 participants: Task completion rate, time-on-task data for 3 tasks, think-aloud quotes, and a written report with severity-rated findings and recommendations.
  • Published UX case study: On Behance or Notion, publicly accessible. Must follow the full case study template: problem → research → insight → ideation → design → testing → outcome.

Grading Rubric

CriterionMarksWhat Examiners Look For
Design System20Tokens documented, components with full variants and anatomy labels, usage guidelines written, consistent application across all screens
Prototype Quality20All major flows clickable, realistic content used, appropriate animations/transitions, error states included
Developer Handoff20Spacing documented with grid overlay, assets named and export-ready, component annotations clear enough for a developer to build without asking questions
Accessibility20WCAG AA across all screens verified with Stark, report documents findings, all issues fixed with before/after evidence
Case Study20Published publicly, follows full template structure, shows real process (research through testing), outcome stated with evidence
Total100Pass mark: 70. This project should represent your best professional work — treat it as a submission to your first client.
