UI/UX Design Quizzes

Five weekly knowledge checks covering UX foundations, research methods, information architecture, visual design, and Figma. Complete each week's content before attempting the quiz.

Week 1 — UX Foundations

5 Questions

1. UX Design primarily focuses on:

UX encompasses the full experience: usability, accessibility, emotional response, and how a product solves a real problem — not just visual appearance.

2. The Double Diamond process has four phases. The first two are:

Double Diamond: Discover (explore problem broadly), Define (narrow to the right problem), Develop (create solutions), Deliver (implement and launch).

3. Jakob Nielsen's research found that usability testing with ___ users finds approximately 85% of usability issues:

Nielsen's landmark study showed 5 users is the optimal number for qualitative usability testing — enough to find most issues without diminishing returns.

4. Which Nielsen heuristic is violated when a system uses jargon the user doesn't understand?

"Match Between System and Real World" requires using the user's language and concepts, not system-internal or technical terminology.

5. A SUS (System Usability Scale) score above 68 indicates:

SUS interpretation: below 68 = below average, 68–80 = good (above average), 80–90 = excellent, above 90 = best imaginable.

Week 2 — User Research

5 Questions

1. An affinity diagram helps to:

Affinity mapping clusters raw observations and quotes from research sessions into emergent themes, revealing patterns across multiple participants.

2. Open-ended interview questions are important in UX research because they:

Open questions like "Tell me about the last time you booked a train ticket" surface stories, workarounds, and emotional responses that closed questions like "Did you find it easy?" never would.

3. Card sorting is primarily used to inform:

Card sorting reveals how users mentally organise and categorise content — directly informing navigation labels, menu groupings, and information hierarchy.

4. A think-aloud protocol in usability testing asks participants to:

Think-aloud reveals what users are thinking at each step — "I'm not sure what this button does" — surfacing confusion before it manifests as task failure.

5. Quantitative research is best for answering:

Quantitative research answers: how many, how often, how fast, how much. Qualitative research answers: why, what motivates, how users think about things.

Week 3 — Information Architecture

5 Questions

1. A sitemap in UX design shows:

A sitemap is a structural diagram showing all screens and how they are connected hierarchically — essentially a blueprint of the product's information architecture.

2. In card sorting, an "open card sort" means:

Open card sort: users create their own categories (reveals mental models). Closed card sort: categories are predefined (validates proposed navigation).

3. "Use the user's language, not internal jargon" is a core principle of:

Navigation labels must match the words users actually use — not internal product names, not technical terms, not marketing-speak. Card sorting surfaces this language.

4. A user flow diagram shows:

A user flow maps screens + decision points for one specific task (e.g. "purchase a product"). It differs from a sitemap, which shows the full structure, not a single task path.

5. Bottom tab navigation is most appropriate when:

Bottom tabs work for 3–5 top-level destinations accessed frequently. More than 5 requires a drawer. Rarely-used sections belong in a hamburger menu.

Week 4 — Visual Design

5 Questions

1. The Gestalt proximity principle states:

Proximity: close things belong together. This is why a label must be closer to its input than to the next label — or users will associate it with the wrong field.

2. WCAG 2.1 AA minimum contrast ratio for normal body text is:

WCAG AA: 4.5:1 for normal text (below 18px or 14px bold); 3:1 for large text (18px+ or 14px+ bold); 7:1 is the AAA standard for highest accessibility.

3. The 8pt grid system in UI design means:

The 8pt grid ensures visual rhythm and consistency — padding, margins, component sizes, and spacing all in multiples of 8 (or 4 for tighter sub-grids).

4. Whitespace in UI design primarily:

Whitespace is an active design element — it creates focus, groups related elements, separates unrelated ones, and prevents visual overwhelm. Cramped UIs increase cognitive load and errors.

5. Serif typefaces (like Georgia or Times New Roman) are generally considered:

Serifs add reading rhythm for long text at appropriate sizes. Medium.com and many editorial sites use serif body text effectively. At small sizes on low-res screens, sans-serif is clearer — context determines the choice.

Week 5 — Figma Basics

5 Questions

1. Figma Auto Layout makes a frame behave like:

Auto Layout mimics CSS flexbox — items stack horizontally or vertically, spacing adjusts automatically, and the container resizes when content changes. Essential for responsive component design.

2. A Figma Component instance:

Component instances inherit changes from the master (like changing border radius globally). But you can also override specific properties on individual instances without breaking the link.

3. Figma Variants allow you to:

Variants group related component states (Default/Hover/Disabled) into one set. In the instance panel, you switch states with a dropdown — no hunting across pages for the right version.

4. The Figma keyboard shortcut to apply Auto Layout is:

Shift+A applies Auto Layout on any selected frame or group of elements. Ctrl+A selects all elements on the canvas — a common confusion.

5. Figma Colour Styles allow you to:

Colour Styles (and Text Styles, Effect Styles) are the Figma equivalent of CSS variables — change the style once and every element using it updates instantly across the entire file.
