4 Labs (Weeks 9–12) TypeScript · Jest · GitHub Actions · Docker Advanced Level
Lab 9 — Migrate Task Manager to TypeScript (Week 9)
1

Add TypeScript to the existing React project

npm install -D typescript @types/react @types/react-dom
npx tsc --init

Rename files: App.jsx → App.tsx, TaskItem.jsx → TaskItem.tsx

2

Define TypeScript interfaces

Create src/types.ts:

export interface Task {
  id: number;
  title: string;
  done: boolean;
}

export interface TaskItemProps {
  task: Task;
  onToggle: (id: number) => void;
  onDelete: (id: number) => void;
}
3

Update components with types

import { Task, TaskItemProps } from '../types';

const TaskItem: React.FC<TaskItemProps> = ({ task, onToggle, onDelete }) => {
  return (
    <div className={`task-item ${task.done ? 'done' : ''}`}>
      <span>{task.title}</span>
      <button onClick={() => onToggle(task.id)}>Done</button>
      <button onClick={() => onDelete(task.id)}>Delete</button>
    </div>
  );
};
4

Fix all TypeScript errors

Run npx tsc --noEmit. Fix every error TypeScript reports. No errors = Lab 9 complete. Common fixes: add return types to functions, type event handlers, handle null with if checks.

Lab 10 — Write Tests with Jest + RTL (Week 10)
1

Install testing tools

npm install -D jest @testing-library/react @testing-library/jest-dom \
  @testing-library/user-event jest-environment-jsdom \
  babel-jest @babel/core @babel/preset-env @babel/preset-react @babel/preset-typescript

Add to package.json:

"scripts": {
  "test": "jest",
  "test:coverage": "jest --coverage"
},
"jest": {
  "testEnvironment": "jsdom",
  "setupFilesAfterEnv": ["@testing-library/jest-dom"]
}
2

Write a utility function test

Create src/utils/taskUtils.ts:

export function countRemaining(tasks: { done: boolean }[]): number {
  return tasks.filter(t => !t.done).length;
}

export function filterByStatus(tasks: { done: boolean }[], showDone: boolean) {
  return tasks.filter(t => t.done === showDone);
}

Create src/utils/taskUtils.test.ts:

import { countRemaining, filterByStatus } from './taskUtils';

const tasks = [
  { id: 1, title: 'A', done: false },
  { id: 2, title: 'B', done: true },
  { id: 3, title: 'C', done: false },
];

describe('countRemaining', () => {
  test('counts undone tasks', () => {
    expect(countRemaining(tasks)).toBe(2);
  });
  test('returns 0 when all done', () => {
    expect(countRemaining([{ done: true }])).toBe(0);
  });
});

describe('filterByStatus', () => {
  test('returns only done tasks', () => {
    expect(filterByStatus(tasks, true).length).toBe(1);
  });
});

Run npm test — all tests must pass.

3

Write a component test

import { render, screen, fireEvent } from '@testing-library/react';
import TaskItem from '../components/TaskItem';

const mockTask = { id: 1, title: 'Test task', done: false };
const mockToggle = jest.fn();
const mockDelete = jest.fn();

test('renders task title', () => {
  render(<TaskItem task={mockTask} onToggle={mockToggle} onDelete={mockDelete} />);
  expect(screen.getByText('Test task')).toBeInTheDocument();
});

test('calls onToggle when Done clicked', () => {
  render(<TaskItem task={mockTask} onToggle={mockToggle} onDelete={mockDelete} />);
  fireEvent.click(screen.getByText('Done'));
  expect(mockToggle).toHaveBeenCalledWith(1);
});
4

Check coverage

Run npm run test:coverage. Coverage report appears in terminal and coverage/ folder. Aim for ≥70% statements and lines. Screenshot the coverage table.

Lab 11 — GitHub Actions CI/CD Pipeline (Week 11)
1

Create the workflow file

Create .github/workflows/ci.yml in your project root:

name: CI — Task Manager

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    name: Test & Build
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js 20
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: TypeScript check
        run: npx tsc --noEmit

      - name: Run tests
        run: npm test -- --watchAll=false --coverage

      - name: Build
        run: npm run build

      - name: Upload coverage report
        uses: actions/upload-artifact@v4
        with:
          name: coverage
          path: coverage/
2

Push and watch the pipeline run

git add .github/
git commit -m "Add CI pipeline"
git push

Go to your GitHub repo → Actions tab. Watch the workflow run. All steps must pass (green checkmarks). Screenshot the successful pipeline run.

3

Add branch protection

GitHub repo → Settings → Branches → Add rule for main. Enable: "Require status checks to pass before merging" → select your CI job. Now main can only receive code that passes all tests.

4

Submission

Submit a link to your GitHub Actions workflow run showing all green steps. This demonstrates industry-standard CI/CD practices.

Lab 12 — Production Polish & Docker (Week 12)
1

Dockerize the Express API

Create Dockerfile in task-api/:

FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --production

COPY . .

EXPOSE 3001
CMD ["node", "index.js"]

Create .dockerignore:

node_modules
.env
*.md

Build and run:

docker build -t task-api .
docker run -p 3001:3001 --env-file .env task-api
2

Run Lighthouse audit on your frontend

Open your deployed Vercel URL in Chrome → F12 → Lighthouse tab → Analyze page load. Review all 4 scores. Fix the top 2 suggestions in the Performance or Accessibility sections. Common quick wins:

  • Add alt to all images
  • Add <meta name="description">
  • Lazy load images below the fold
3

Final portfolio check

Your completed Domain 1 portfolio must include:

  • Live portfolio website on GitHub Pages
  • Full-stack Task Manager live on Vercel + Render
  • GitHub repo with CI pipeline passing
  • Test coverage ≥ 70%
  • TypeScript throughout
  • Lighthouse Performance score ≥ 80
4

Final Submission

Submit to your mentor: GitHub repos links, live deployment URLs, screenshot of CI pipeline passing, screenshot of Lighthouse report.

Deliverable: Complete production-ready project
