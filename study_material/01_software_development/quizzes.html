Week 1 Quiz — HTML5 Foundations

1. Which HTML element is used for the primary navigation links on a page?

A) <menu>
B) <nav>
C) <header>
D) <links>
The <nav> element is a semantic HTML5 element specifically for navigation links. <header> is for the top section of a page, and <menu> is rarely used. Using semantic tags helps screen readers and search engines understand your page structure.

2. What does DOCTYPE in <!DOCTYPE html> tell the browser?

A) The page uses CSS3
B) The server type
C) This is an HTML5 document
D) The JavaScript version
<!DOCTYPE html> tells the browser to render the page in HTML5 standards mode. Without it, the browser may enter "quirks mode" and render differently. It must always be the very first line in your HTML file.

3. Which attribute is REQUIRED on every <img> tag for accessibility?

A) alt
B) title
C) width
D) id
The alt attribute provides alternative text for screen readers and when images fail to load. It is required for Web Content Accessibility Guidelines (WCAG) compliance and is also used by Google for image SEO ranking.

4. Where should CSS <link> tags be placed in an HTML document?

A) At the end of <body>
B) After the </html> tag
C) Inside <footer>
D) Inside <head>
CSS <link> tags go in <head>. This allows the browser to load and parse CSS before rendering the page, preventing a "flash of unstyled content" (FOUC). Scripts usually go before </body>, but CSS goes in <head>.

5. What is the difference between <div> and <section>?

A) <div> is for mobile, <section> for desktop
B) <section> has semantic meaning, <div> is purely for layout
C) They are exactly the same
D) <div> cannot have CSS applied to it
<section> is a semantic element representing a thematic grouping of content — screen readers and search engines understand it means a meaningful section. <div> has zero semantic meaning and should only be used for layout/styling purposes when no semantic element fits.
Week 2 Quiz — CSS3 & Flexbox

1. In the CSS Box Model, what is the correct order from inside to outside?

A) Margin → Border → Padding → Content
B) Content → Border → Padding → Margin
C) Content → Padding → Border → Margin
D) Padding → Content → Border → Margin
The Box Model from inside out: Content (your text/image) → Padding (space inside the border) → Border (the line) → Margin (space outside the border). Remember it as: C-P-B-M or "Can Penguins Breathe Mars?"

2. Which CSS property controls the horizontal alignment of flex items?

A) justify-content
B) align-items
C) flex-direction
D) flex-wrap
justify-content aligns flex items along the main axis (horizontal by default). align-items aligns them on the cross axis (vertical by default). Remember: J = horizontal, A = vertical for the default row direction.

3. How do you define a CSS custom property (variable)?

A) $color-navy: #1B3A6B;
B) @variable color-navy: #1B3A6B;
C) color-navy = #1B3A6B;
D) --color-navy: #1B3A6B;
CSS custom properties use the -- prefix and are typically defined inside :root { } for global scope. The $ syntax is SCSS (a preprocessor), not standard CSS.

4. Why is box-sizing: border-box recommended?

A) It makes all elements the same width
B) Width includes padding and border, making sizing predictable
C) It removes all margins
D) It forces all elements to display as block
By default (content-box), a 200px wide element with 20px padding becomes 240px total. With border-box, the 200px includes padding and border, so the element stays 200px wide. This makes layouts far more predictable.

5. Which font unit is best for accessibility?

A) px (pixels)
B) pt (points)
C) rem (root em)
D) cm (centimetres)
rem (root em) is relative to the browser's root font size. If a user increases their browser font size for accessibility, rem-based layouts scale accordingly. px ignores this setting, which can make sites unusable for visually impaired users.
Week 3 Quiz — Responsive Design & Grid

1. In mobile-first design, which type of media query is used to scale UP to larger screens?

A) @media (max-width: 768px)
B) @media (min-width: 768px)
C) @media screen only
D) @media (device: tablet)
Mobile-first means you write base styles for mobile (no query), then use min-width queries to add styles for larger screens. max-width is desktop-first (opposite approach), which is less efficient.

2. What does "1fr" mean in CSS Grid?

A) 1 fraction of the available free space
B) 1 fixed pixel
C) 1 percent of the viewport
D) 1 font-size unit
fr is the "fraction unit" — it represents a portion of the available space in the grid container. repeat(3, 1fr) creates 3 equal columns, each taking one-third of the available space.

3. Which meta tag is REQUIRED for responsive design to work on mobile browsers?

A) <meta name="mobile" content="responsive">
B) <meta name="screen" content="fit">
C) <meta name="viewport" content="width=device-width, initial-scale=1.0">
D) <meta name="responsive" content="true">
Without the viewport meta tag, mobile browsers render the page at desktop width (980px typically) and scale it down, making everything tiny. This meta tag tells the browser to use the actual device width and set zoom to 1.

4. When should you use CSS Grid vs Flexbox?

A) Grid is for old browsers, Flexbox is modern
B) Flexbox for 2D layouts, Grid for 1D
C) They cannot be used together
D) Grid for 2D layouts (rows + columns), Flexbox for 1D (row OR column)
Use CSS Grid when you need control over both rows AND columns simultaneously (page layouts, card grids). Use Flexbox when items flow in one direction (navigation bars, button groups, centering a single item).

5. What does repeat(auto-fill, minmax(250px, 1fr)) do in CSS Grid?

A) Creates as many columns as fit, each at least 250px wide, responsive without media queries
B) Creates exactly 250 columns
C) Makes every item 1 pixel wide
D) Fills the grid with 250 rows
auto-fill creates as many columns as fit the container. minmax(250px, 1fr) means each column is at least 250px but can grow. Together they create a fully responsive grid — no media queries needed. It's one of the most powerful patterns in CSS.
Week 4 Quiz — JavaScript Basics & DOM

1. What is the correct way to declare a variable whose value will not change?

A) var name = "Priya";
B) let name = "Priya";
C) const name = "Priya";
D) fixed name = "Priya";
Use const for values that won't be reassigned. Use let when you need to change the value later. Never use var in modern JavaScript — it has function scope and hoisting behavior that causes bugs.

2. How do you select an element with class "btn-primary" using JavaScript?

A) document.getElement('.btn-primary')
B) document.querySelector('.btn-primary')
C) document.findElement('btn-primary')
D) document.selectElement('.btn-primary')
querySelector takes any CSS selector string. For a class, use .classname. For an ID, use #idname. It returns the first match. Use querySelectorAll to get all matching elements as a NodeList.

3. What does event.preventDefault() do in a form submit handler?

A) Stops the page from reloading when the form submits
B) Clears all form inputs
C) Removes the event listener
D) Submits the form to the server automatically
By default, submitting a form causes a page reload (HTTP GET/POST). preventDefault() cancels this browser default, allowing you to handle the submission with JavaScript instead — which is required for single-page applications.

4. Which is the correct way to add a click event listener?

A) button.on('click', handler)
B) button.onclick = handler()
C) <button onclick="handler()"> in HTML
D) button.addEventListener('click', handler)
addEventListener is the modern, correct way. It allows multiple listeners on the same element and keeps JavaScript separate from HTML. Option C (inline onclick) mixes concerns and is bad practice. Option B passes the function's return value, not the function itself.

5. What does element.classList.toggle('active') do?

A) Removes all classes from the element
B) Always adds the 'active' class
C) Adds 'active' if absent, removes it if present
D) Checks if 'active' class exists (returns true/false)
toggle is a convenient method that flips a class on/off. Use it for hamburger menus, dark mode toggles, and accordion panels. add() always adds, remove() always removes, contains() checks — toggle does both.
Week 5 Quiz — React.js Fundamentals

1. In React, what must all component names start with?

A) A lowercase letter
B) An uppercase letter
C) The word "React"
D) "use" (like hooks)
React components must start with a capital letter. This is how React distinguishes between custom components (<TaskItem />) and native HTML elements (<div>). A lowercase component name like <taskItem /> would be treated as an unknown HTML tag.

2. What does setCount(count + 1) do in React state?

A) Updates state and triggers a re-render of the component
B) Directly modifies the count variable
C) Stores the value to localStorage
D) Sends the value to the parent component
Calling the setter function (setCount) does two things: updates the state value AND schedules a re-render of the component with the new value. React's rendering is driven by state changes — this is the fundamental mechanism.

3. When should useEffect with an empty dependency array [] be used?

A) To run on every render
B) To never run the effect
C) To run when any state changes
D) To run only once when the component mounts
useEffect(fn, []) runs once after the initial render ("on mount"). It's perfect for initial API calls, setting up subscriptions, or reading localStorage. Without the [], it runs after every render, which usually causes infinite loops.

4. Why does every list item in React need a unique "key" prop?

A) For CSS styling purposes
B) To set the item's ID in the DOM
C) To help React identify which items changed, were added, or removed efficiently
D) It is optional and only needed for large lists
React's Virtual DOM uses keys to match old and new lists efficiently. Without stable keys, React re-renders the entire list on any change. Use the item's unique ID as the key, not the array index — index keys cause subtle bugs when items are reordered.

5. In JSX, which attribute replaces the HTML "class" attribute?

A) className
B) classId
C) classList
D) cssClass
In JSX, class is a reserved keyword in JavaScript (used for ES6 classes), so React uses className instead. Similarly, for becomes htmlFor, and tabindex becomes tabIndex — JSX uses camelCase for HTML attributes.
