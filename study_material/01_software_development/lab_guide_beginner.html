Beginner Lab Guide

4 hands-on lab exercises — one per week. Build a real portfolio website from scratch. Each lab builds on the previous one. By Week 4 you will have a live, deployed website.

4 Labs VS Code · Chrome DevTools · GitHub Beginner Level

Lab Setup — Do This Once Before Lab 1

  • Install VS Code from code.visualstudio.com
  • Install the "Live Server" extension in VS Code
  • Create a free account at github.com
  • Install Git from git-scm.com
  • Configure Git: git config --global user.name "Your Name" and git config --global user.email "you@email.com"
Lab 1 — Build Your Personal Portfolio Page (Week 1)
1

Create your project folder

Create a folder on your Desktop called my-portfolio. Open VS Code → File → Open Folder → select your folder. Create a new file called index.html.

2

Start with the HTML skeleton

Type ! and press Tab in VS Code — it generates the full HTML5 boilerplate. Update the title to your name. Then add this structure inside <body>:

<header>
  <nav>
    <a href="#about">About</a>
    <a href="#skills">Skills</a>
    <a href="#contact">Contact</a>
  </nav>
</header>

<main>
  <section id="about">
    <h1>Hello, I'm [Your Name]</h1>
    <p>BCA student at [Your College], passionate about web development.</p>
  </section>

  <section id="skills">
    <h2>Skills</h2>
    <ul>
      <li>HTML5</li>
      <li>CSS3</li>
      <li>JavaScript</li>
    </ul>
  </section>

  <section id="contact">
    <h2>Contact Me</h2>
    <form>
      <label for="name">Your Name:</label>
      <input type="text" id="name" placeholder="Enter your name">
      <label for="msg">Message:</label>
      <textarea id="msg" placeholder="Write your message..."></textarea>
      <button type="submit">Send Message</button>
    </form>
  </section>
</main>

<footer>
  <p>© 2026 [Your Name]</p>
</footer>
3

Open in Live Server

Right-click on index.html in VS Code → "Open with Live Server." Your page opens in the browser. Every time you save, it refreshes automatically. Verify all sections are visible.

4

Validate your HTML

Go to validator.w3.org. Select "Validate by Direct Input." Paste your HTML. Fix any errors shown. Zero errors = Lab 1 complete.

5

Submission

Take a screenshot of your browser showing the page AND the validator showing zero errors. Upload both to the submission portal.

Deliverable: screenshot + clean validation
Lab 2 — Style with CSS3 + Flexbox (Week 2)
1

Create style.css

Create style.css in the same folder. Link it in your HTML head: <link rel="stylesheet" href="style.css">. Start with the CSS reset and variables:

:root {
  --navy:    #1B3A6B;
  --gold:    #C8941A;
  --cream:   #F8F6F0;
  --white:   #ffffff;
  --font:    'Segoe UI', sans-serif;
}

*, *::before, *::after {
  margin: 0; padding: 0;
  box-sizing: border-box;
}

body {
  font-family: var(--font);
  background: var(--cream);
  color: #333;
  line-height: 1.6;
}
2

Style the navigation

header {
  background: var(--navy);
  padding: 16px 32px;
  position: sticky;
  top: 0;
  z-index: 100;
}

nav {
  display: flex;
  gap: 24px;
  justify-content: flex-end;
}

nav a {
  color: white;
  text-decoration: none;
  font-size: 14px;
  font-weight: 600;
  letter-spacing: .5px;
}

nav a:hover {
  color: var(--gold);
}
3

Style the About section as a hero

#about {
  background: var(--navy);
  color: white;
  text-align: center;
  padding: 80px 32px;
}

#about h1 {
  font-size: 2.5rem;
  margin-bottom: 16px;
}

#about p {
  font-size: 1.1rem;
  opacity: 0.85;
  max-width: 600px;
  margin: 0 auto;
}
4

Use Chrome DevTools to experiment

Press F12 in Chrome. Click the element picker (top-left icon). Click any element on your page. Try changing CSS values in the Styles panel on the right. This is temporary — copy good values back to your file.

5

Submission

Your styled page should have: sticky nav, hero section, styled skills list, and a styled form. Screenshot the full page.

Deliverable: styled portfolio screenshot
Lab 3 — Responsive Design with CSS Grid (Week 3)
1

Convert skills list to a responsive grid

Add more skills to your list and wrap them in a grid container:

/* HTML — replace the ul with this */
<div class="skills-grid">
  <div class="skill-card">HTML5</div>
  <div class="skill-card">CSS3</div>
  <div class="skill-card">JavaScript</div>
  <div class="skill-card">Responsive Design</div>
  <div class="skill-card">Git</div>
  <div class="skill-card">VS Code</div>
</div>

/* CSS */
.skills-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(140px, 1fr));
  gap: 16px;
  padding: 32px;
}

.skill-card {
  background: var(--navy);
  color: white;
  text-align: center;
  padding: 20px;
  border-radius: 12px;
  font-weight: 700;
  font-size: 14px;
}
2

Test responsiveness with DevTools

Press F12 → click the phone/tablet icon (Toggle Device Toolbar). Test at: 360px (Android), 414px (iPhone), 768px (iPad), 1280px (laptop). Your grid should adjust at every size without content overflowing.

3

Add a media query for mobile nav

@media (max-width: 600px) {
  nav {
    justify-content: center;
    gap: 12px;
    flex-wrap: wrap;
  }

  #about h1 {
    font-size: 1.8rem;
  }
}
4

Submission

Submit two screenshots: desktop view (1280px) and mobile view (360px) from DevTools.

Deliverable: desktop + mobile screenshots
Lab 4 — JavaScript + Deploy to GitHub Pages (Week 4)
1

Create script.js and make the form interactive

// script.js
const form = document.querySelector('#contact form');
const nameInput = document.querySelector('#name');
const msgInput = document.querySelector('#msg');

form.addEventListener('submit', (e) => {
  e.preventDefault();

  const name = nameInput.value.trim();
  const msg  = msgInput.value.trim();

  if (!name) {
    alert('Please enter your name');
    nameInput.focus();
    return;
  }

  if (!msg) {
    alert('Please enter a message');
    msgInput.focus();
    return;
  }

  // Show success message
  form.innerHTML = `
    <div style="text-align:center; padding:32px;">
      <h3 style="color:#2E7D32;">✓ Message sent!</h3>
      <p>Thank you, ${name}. I'll get back to you soon.</p>
    </div>
  `;
});

Link this file in your HTML: <script src="script.js"></script> (before closing </body>)

2

Push to GitHub

Open VS Code Terminal (Ctrl+`):

git init
git add .
git commit -m "Add portfolio website"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/YOUR-USERNAME.github.io.git
git push -u origin main

Replace YOUR-USERNAME with your actual GitHub username (repository must be named username.github.io).

3

Enable GitHub Pages

On GitHub: Settings → Pages → Source: Deploy from branch → Branch: main → Save. Wait 2–3 minutes. Your site will be live at https://YOUR-USERNAME.github.io.

4

Submission

Submit the live URL of your deployed portfolio. It must show all three sections (About, Skills, Contact) with working JavaScript form validation.

Deliverable: live GitHub Pages URL
