Skill Milestones — Track Your Progress

🌱 Foundational (Weeks 1–4)

  • Build a static webpage from scratch
  • Use Flexbox and Grid for layouts
  • Make a page fully responsive
  • Add JavaScript interactivity
  • Deploy to GitHub Pages

⚡ Application (Weeks 5–8)

  • Build and run a React app with hooks
  • Create a REST API with Express
  • Connect frontend to a live database
  • Handle CORS, auth, and env variables
  • Deploy both frontend and backend

🚀 Professional (Weeks 9–12)

  • TypeScript codebase with zero errors
  • Test coverage ≥ 70%
  • CI/CD pipeline on GitHub Actions
  • Docker container for backend
  • Lighthouse score ≥ 80

💼 Job-Ready Checklist

  • 3+ projects on live URLs in portfolio
  • GitHub profile active (commits, green)
  • LinkedIn with projects and skills
  • Resume reviewed by mentor
  • 2+ mock interviews completed
90-Day Post-Internship Action Plan
1

Days 1–30: Polish & Publish

Make your internship work public and visible

  • Deploy all 3 projects to live URLs (Vercel/Render/GitHub Pages)
  • Write a README.md for each GitHub repo explaining the project, tech stack, and how to run it
  • Update LinkedIn: add internship under Experience, list all projects, add skills
  • Pin your best 3 repos on GitHub profile
  • Create a 1-page resume focused on projects and skills (use a clean ATS-friendly template)
  • Complete 1 certification (freeCodeCamp JavaScript is recommended)
2

Days 31–60: Apply & Practice

Start applications while sharpening interview skills

  • Apply to 5–10 junior developer roles per week on LinkedIn, Naukri, Internshala
  • Solve 1 LeetCode Easy problem per day (focus: arrays, strings, objects)
  • Do 2 mock interviews with peers from this program
  • Build one small new project that shows a skill gap you identified
  • Reach out to 5 alumni/professionals on LinkedIn asking for a 15-min call
  • Attend at least 1 local or online tech meetup
3

Days 61–90: Interview & Negotiate

Convert applications to offers

  • Prepare answers for the 10 most common frontend interview questions
  • Practice system design basics: explain your blog platform architecture
  • Research each company before interview: their tech stack, products, team size
  • Know your numbers: minimum salary, target salary, location preferences
  • Follow up with every recruiter within 24 hours of any interaction
  • Do not stop applying until an offer letter is signed
