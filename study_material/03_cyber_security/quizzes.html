Week 1 Quiz — The CIA Triad & Security Fundamentals

1. What does CIA stand for in cyber security?

A) Central Intelligence Agency
B) Confidentiality, Integrity, Availability
C) Control, Inspect, Audit
D) Cyber Incident Analysis
CIA Triad is the foundational model of information security. Confidentiality = only authorized users see data. Integrity = data hasn't been tampered with. Availability = system is accessible when needed. Every security decision maps to one or more of these three pillars.

2. Which threat actor is typically motivated by political or ideological causes?

A) Script kiddie
B) Cybercriminal
C) Hacktivist
D) Insider threat
Hacktivists (e.g. Anonymous) hack for ideological or political reasons. Cybercriminals are financially motivated. Script kiddies use existing tools for fun or notoriety. Insider threats have legitimate internal access and may be motivated by grievance or financial incentive.

3. What is Defense in Depth?

A) Relying on one strong security control
B) Multiple layers of security controls so one failure doesn't compromise everything
C) Deep background checks on all employees
D) Defending the database layer first
Defense in Depth means if one control fails (e.g. the firewall is bypassed), others still protect the system (e.g. endpoint detection, network segmentation, MFA). No single control is 100% reliable — layering is what makes a system resilient.

4. What is an "attack surface"?

A) The physical screen of a compromised computer
B) All possible points where an attacker could enter or extract data from a system
C) The space behind a server rack
D) The visible user interface of a web application
Attack surface = every login page, API endpoint, employee, physical door, USB port, and third-party integration. Reducing attack surface (disabling unused services, removing unused accounts, closing unnecessary ports) is a primary security strategy.

5. Which property is NOT part of the CIA Triad?

A) Confidentiality
B) Integrity
C) Authentication
D) Availability
Authentication is important but not part of the CIA Triad. It is a mechanism that supports Confidentiality (by ensuring only the right people can access data), but it is not itself a pillar of the triad. The three pillars are Confidentiality, Integrity, and Availability.
Week 2 Quiz — Common Attack Types

1. What is spear phishing?

A) Generic mass phishing emails sent to thousands of people
B) Targeted phishing at a specific individual or organization with personalized content
C) Phishing conducted over phone calls
D) Phishing conducted via SMS messages
Spear phishing is personalized — attackers research their target (name, role, colleagues, recent company news) to make the email convincing. It is far more effective than generic mass phishing and is the technique used in most major corporate breaches.

2. Which type of malware encrypts your files and demands payment to restore access?

A) Spyware
B) Adware
C) Trojan
D) Ransomware
Ransomware (e.g. WannaCry, LockBit, REvil) encrypts victim files making them inaccessible, then demands cryptocurrency payment for the decryption key. Regular offline backups are the primary defense — they allow recovery without paying the ransom.

3. What is "pretexting" in social engineering?

A) Reading the fine print before signing a contract
B) Creating a fabricated scenario to manipulate someone into giving up information
C) Testing systems with pre-defined test data
D) Setting up a honeypot to attract attackers
Pretexting = inventing a believable story to gain trust and extract information. Example: calling the IT helpdesk pretending to be a new employee who forgot their password. Training staff and having robust verification procedures defend against it.

4. DDoS attacks primarily target which component of the CIA Triad?

A) Confidentiality
B) Integrity
C) Availability
D) Authentication
DDoS (Distributed Denial of Service) floods servers with traffic from many sources, making them unavailable to legitimate users — a direct attack on Availability. It does not steal or modify data; it simply prevents access to the service.

5. What is the BEST way to check if a link in an email is safe before clicking it?

A) Click it quickly and immediately close the tab
B) Hover over the link to reveal the real URL in the browser status bar without clicking
C) Forward the email to a friend to test for you
D) Search for the company name in Google to find the real site
Hovering reveals the actual destination URL in the browser status bar. The visible text of a link can say anything — it's the href destination that matters. If the actual URL domain doesn't match the claimed sender's domain, it's phishing.
Week 3 Quiz — Network Security Basics

1. What does a stateful firewall track that a stateless firewall does not?

A) User passwords
B) The state of active network connections
C) File access logs
D) CPU usage of connected devices
A stateful firewall tracks active connections — it knows if a packet is part of an established, expected connection or a new unexpected one. This lets it block unexpected inbound packets even if they use an allowed port. Stateless firewalls only check each packet in isolation against static rules.

2. What does HTTPS provide that HTTP does not?

A) Faster page loading speeds
B) More colorful website rendering
C) Encryption of data in transit between browser and server
D) Better SEO ranking automatically
HTTPS adds TLS encryption to HTTP, preventing network eavesdroppers from reading or modifying data in transit. HTTP sends everything — including passwords and session tokens — as readable plain text. Anyone on the same network (café WiFi, ISP) can intercept it.

3. What is the primary security purpose of a VPN?

A) Speed up internet connection
B) Block advertisements on websites
C) Encrypt traffic between your device and the VPN server, protecting it from local interception
D) Bypass download speed limits imposed by ISPs
A VPN creates an encrypted tunnel for your traffic. This is especially important on public WiFi where an attacker on the same network could perform a man-in-the-middle attack to intercept unencrypted communications. Note: VPN protects you from the local network, not from the VPN provider itself.

4. What does asymmetric encryption use?

A) The same key for both encrypting and decrypting
B) A public key to encrypt and a separate private key to decrypt
C) A shared secret negotiated between both parties in advance
D) A physical hardware key card
Asymmetric encryption uses a key pair: the public key (shared with anyone) encrypts data; only the private key (kept secret) can decrypt it. HTTPS uses this to securely exchange a symmetric session key, then switches to faster symmetric encryption for the session data.

5. You see a padlock icon in your browser address bar. What does this confirm?

A) The website is safe and can be trusted completely
B) The website is a legitimate business and not a scam
C) Traffic between your browser and the server is encrypted with TLS
D) The website has been scanned and contains no malware
The padlock only confirms TLS encryption — it does NOT verify the site is legitimate, safe, or run by who it claims. Phishing sites routinely obtain TLS certificates (they're free via Let's Encrypt). Never trust a site just because it has a padlock — check the domain name carefully.
Week 4 Quiz — Account Security & Safe Practices

1. What is the minimum recommended password length according to NIST 2024 guidelines?

A) 6 characters
B) 8 characters
C) 12 characters
D) 16+ characters
NIST SP 800-63B (2024 revision) recommends 15+ characters minimum. Length beats complexity — "correct-horse-battery-staple" (28 chars) is exponentially stronger than "P@ssw0rd!" (9 chars). Use a password manager to generate and store long random passwords so you don't have to remember them.

2. Which type of MFA is considered the most secure for everyday accounts?

A) SMS one-time passwords (OTP)
B) Email one-time passwords (OTP)
C) TOTP authenticator app (e.g. Google Authenticator, Aegis)
D) Security questions
TOTP apps generate codes locally on your device — they cannot be intercepted via SS7 attacks (which affect SMS) or email compromise. Security questions are actually weaker than a password — they're guessable and publicly researched. Hardware keys (YubiKey) are even stronger than TOTP for critical accounts.

3. What is the Principle of Least Privilege?

A) Use the least expensive security software available
B) Grant users only the minimum access they need to do their specific job
C) Use the fewest number of security controls possible
D) Hire the least experienced staff to save budget
Least Privilege limits the blast radius when an account is compromised. A database analyst only needs SELECT, not DROP TABLE. An intern shouldn't have admin rights. If a low-privilege account is breached, the attacker's access is limited. Review and remove unnecessary access regularly — access creep is a real risk.

4. According to Microsoft's research, what percentage of automated account attacks does MFA block?

A) 40%
B) 65%
C) 80%
D) 99.9%
Microsoft's research (confirmed in 2019 and 2023 reports) shows MFA blocks 99.9% of automated credential-stuffing and password-spray attacks. This makes it the single highest-impact security control for user accounts. The remaining 0.1% are targeted attacks that specifically bypass MFA (e.g. real-time phishing proxies).

5. What does a password manager primarily do?

A) A person employed to manage passwords for an organization
B) Software that generates cryptographically random passwords and stores them encrypted
C) A secure text file where you write down all your passwords
D) A browser feature that auto-fills saved passwords
Password managers (Bitwarden, 1Password) generate random passwords and store them encrypted. You only need to remember one master passphrase. They also detect phishing sites by checking if the saved domain matches the current URL — they won't autofill credentials on a fake site.
Week 5 Quiz — OWASP Top 10

1. What is SQL Injection?

A) Physically inserting a needle into a SQL server
B) Injecting malicious SQL code into input fields that gets executed by the database
C) A type of database backup procedure
D) Adding new valid SQL records to a database
SQL Injection occurs when user input is directly embedded in an SQL query. Input like ' OR '1'='1 can bypass login entirely. Parameterized queries (prepared statements) prevent it 100% by ensuring user input is treated as data, never as executable SQL code.

2. According to OWASP Top 10 2021, what is the #1 most common web vulnerability category?

A) SQL Injection
B) Cryptographic Failures
C) Broken Access Control
D) Security Misconfiguration
Broken Access Control (A01:2021) moved to #1 because of how widespread IDOR, privilege escalation, and missing authorization checks are. Users accessing data or functions they shouldn't — by simply changing a number in a URL — is the most common class of web vulnerability.

3. What does IDOR stand for?

A) Insecure Database Object Records
B) Insecure Direct Object Reference
C) Internal Data Object Repository
D) Integrated Defense Object Routing
IDOR (Insecure Direct Object Reference) = accessing another user's data by changing an identifier in a request (e.g. changing /user/123/profile to /user/124/profile). The fix: check server-side that the requesting user is authorized to access the specific resource on every request.

4. What is Cross-Site Scripting (XSS)?

A) Copying CSS styles between different websites
B) Injecting malicious JavaScript into web pages that other users view and execute
C) Cross-browser CSS compatibility issues
D) Server-side script injection into database records
XSS (Cross-Site Scripting) injects JavaScript into a page served to other users. Stored XSS is the most dangerous — the script is saved in the database and runs for every visitor automatically. Prevention: output encoding (never insert user data directly into HTML), and Content Security Policy (CSP) headers.

5. Which OWASP category covers using a library or framework with a known, unpatched vulnerability?

A) A01 Broken Access Control
B) A03 Injection
C) A06 Vulnerable and Outdated Components
D) A09 Security Logging and Monitoring Failures
A06 Vulnerable and Outdated Components covers using npm packages, pip libraries, or frameworks with known CVEs. Use npm audit or tools like Snyk to detect them. The 2017 Equifax breach (147 million records exposed) was caused by an unpatched Apache Struts vulnerability — a textbook A06 failure.
