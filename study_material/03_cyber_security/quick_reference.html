Nmap Quick Reference
Nmap — Essential Commands
nmap <target>                    # basic SYN scan
nmap -sV <target>               # version detection
nmap -sC <target>               # default scripts
nmap -sV -sC <target>          # version + scripts
nmap -p 80,443 <target>        # specific ports
nmap -p- <target>              # all 65535 ports
nmap -A <target>               # aggressive: OS+version+scripts
nmap --top-ports 1000 <target> # top 1000 ports
nmap -oN output.txt <target>   # save to file
nmap -Pn <target>              # skip host discovery
Burp Suite Cheat Sheet
  • Proxy: intercept HTTP/HTTPS — configure browser to use 127.0.0.1:8080 as proxy
  • Repeater: modify and resend requests — right-click request → Send to Repeater
  • Intruder: automated fuzzing — Sniper = one parameter, Cluster Bomb = multiple
  • Scanner: automated vuln scanner (Pro only; Community has passive scanning)
  • Spider/Crawler: map all application endpoints and parameters
  • Decoder: encode/decode Base64, URL encoding, HTML entities, hex
Incident Response Checklist
IR Checklist — Keep This Handy
── DETECT ────────────────────────────────────────────
[ ] Alert received from SIEM / user report / external notification
[ ] Initial triage: confirm true positive (not false alarm)
[ ] Assign severity: SEV1=critical / SEV2=high / SEV3=med / SEV4=low

── CONTAIN ────────────────────────────────────────────
[ ] Isolate affected systems from the network (VLAN, firewall rule)
[ ] Preserve evidence FIRST — memory dump, log copies before wiping
[ ] Block malicious IPs and domains at firewall and DNS level
[ ] Notify on-call security lead and management per escalation policy

── ERADICATE ──────────────────────────────────────────
[ ] Identify root cause — how did the attacker get in?
[ ] Remove malware, backdoors, and unauthorized accounts
[ ] Patch the exploited vulnerability
[ ] Reset ALL compromised credentials (not just the obvious ones)

── RECOVER ────────────────────────────────────────────
[ ] Restore from known-clean backup (verify backup integrity first)
[ ] Monitor systems closely for 72 hours after restoration
[ ] Verify normal business operations are fully restored

── LESSONS LEARNED ────────────────────────────────────
[ ] Write incident report (timeline, root cause, impact, cost)
[ ] Conduct blameless postmortem with all stakeholders
[ ] Update detection rules and runbooks
[ ] File CERT-In report if required (within 6 hours — Indian law)

⚠️ Legal Reminder: Tools marked "authorized only" must NEVER be used against systems you do not own or have explicit written permission to test.
