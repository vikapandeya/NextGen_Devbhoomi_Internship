Project Briefs — Cyber Security

Three projects demonstrating security assessment, defense implementation, and incident response skills.

3 Projects Beginner · Intermediate · Advanced Weeks 4 / 8 / 12 Portfolio-Grade Work
Project 1 — Beginner
Project 01 of 03

Personal Security Hardening Report

Conduct a full personal security audit of your digital life and implement measurable improvements. This is your first real security assessment — document the before and after state with evidence.

🌱 Beginner
Research Password Manager MFA Browser Security HaveIBeenPwned

Requirements (Checklist)

  • Check all personal email addresses on HaveIBeenPwned — document each breach found (service, date, data types exposed)
  • Install Bitwarden and migrate a minimum of 20 accounts to strong unique passwords (20+ characters each)
  • Enable MFA on all critical accounts: primary email, all banking/financial apps, GitHub, social media accounts
  • Audit all browser extensions — remove any that are unused, have excessive permissions, or are from unverified publishers
  • Enable full-disk encryption on your laptop (BitLocker on Windows, FileVault on Mac)
  • Configure automatic OS and browser updates — confirm they are active with a screenshot
  • Document before/after state with screenshots for each step (Bitwarden health report, HIBP results, MFA confirmations)
CriteriaPoints
Documentation quality — clear structure, screenshots, before/after evidence20
HIBP audit + Bitwarden setup with 20+ migrated accounts25
MFA enabled on all critical accounts with confirmation evidence25
Browser extension audit + OS hardening (encryption + auto-updates)20
Written reflection — what surprised you, what risk did you eliminate, what remains10
Total  ·  Pass Mark: 60/100100
📅 Deadline: End of Week 4
Project 2 — Intermediate
Project 02 of 03

Web Application Security Assessment

Conduct a structured penetration test against DVWA following the OWASP Testing Methodology. Document all findings in a professional report you could present to a real client.

⚡ Intermediate
DVWA Burp Suite OWASP Methodology CVSS v3.1 Pen Test Report

Requirements (Checklist)

  • Test all applicable OWASP Top 10 categories against DVWA at Low security — document whether each is present or not present
  • Minimum 5 documented vulnerabilities with: CVE reference (where applicable), CVSS v3.1 base score (use the NIST calculator), proof-of-concept reproduction steps, screenshot evidence, and a specific actionable remediation recommendation
  • Professional report with Executive Summary (non-technical, 1 page), Scope and Methodology section, Findings section (one detailed page per finding), Risk Rating summary table, and full screenshot appendix
  • CVSS scores must be justified — list the vector components you chose (AV:N/AC:L/…) and why each was selected
  • Remediation recommendations must be specific — reference the vulnerable function/file/parameter, not just the general fix pattern
CriteriaPoints
Number of valid, documented vulnerabilities (minimum 5 required)25
Quality of proof-of-concept steps — reproducible, precise, with evidence25
CVSS v3.1 scoring accuracy with justification of vector components15
Remediation specificity and technical accuracy20
Report professionalism — structure, language, executive summary quality15
Total  ·  Pass Mark: 65/100100
📅 Deadline: End of Week 8
Project 3 — Advanced
Project 03 of 03

Cloud Security Posture Assessment & Hardening

Conduct a full security assessment of an AWS environment, implement all critical findings, and build an automated security monitoring solution in Python. This is your capstone portfolio piece for Cyber Security.

🚀 Advanced
AWS GuardDuty CloudTrail IAM Python boto3 SNS

Requirements (Checklist)

  • IAM audit: credential report analysis, identify unused users/roles (no login in 90+ days), identify overprivileged policies (AdministratorAccess attached to non-admin users), MFA status for all users
  • S3 audit: check all buckets for public access, versioning status, server-side encryption, and bucket policies — document each finding with risk level
  • Network audit: review Security Groups for overly permissive inbound rules (0.0.0.0/0 on sensitive ports), check VPC Flow Logs are enabled, review NACLs
  • Write a Python script using boto3 that programmatically checks for 5 specific misconfigurations (e.g. MFA not on root, S3 bucket is public, Security Group allows 0.0.0.0/0 on port 22) and outputs a JSON report with findings, severity, and resource ARN
  • Enable and configure GuardDuty + Security Hub — screenshot showing both are active and collecting findings
  • Create an SNS alert that emails you when the root account is used (CloudWatch Event → SNS → Email)
  • Before and after screenshots for every remediation — prove the fix was applied, not just described
  • Write a 15+ page security assessment report following the structure from Project 2 but for cloud infrastructure
CriteriaPoints
IAM, S3, and network audit findings — accuracy, completeness, risk assessment25
boto3 Python script — code quality, handles errors, produces valid JSON output20
GuardDuty + Security Hub setup + SNS root login alert — all three operational with evidence20
Assessment report quality — 15+ pages, findings with evidence, remediation roadmap25
Before/after remediation evidence — each remediation confirmed with screenshots10
Total  ·  Pass Mark: 70/100100
