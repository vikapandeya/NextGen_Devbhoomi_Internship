Week 1 — The CIA Triad & Security Fundamentals
W1

The CIA Triad & Security Fundamentals

Confidentiality · Integrity · Availability · Threat Actors · Attack Surface · Defense in Depth

▼
Analogy: Security is like protecting your home. Confidentiality = window blinds (nobody sees inside). Integrity = door locks (nobody changes what's inside). Availability = you can always get in when you need to.

CIA Triad

  • Confidentiality — only authorized users can access data
  • Integrity — data remains unchanged and accurate
  • Availability — system accessible when needed
  • Every security decision maps to one or more of these three
  • Encryption protects Confidentiality; hashing protects Integrity; redundancy protects Availability

Threat Actors

  • Script kiddies — use existing tools for fun/notoriety, low skill
  • Cybercriminals — financially motivated, organized gangs
  • Nation-states — government-sponsored, highly sophisticated
  • Insiders — employees with legitimate access, high risk
  • Hacktivists — ideological/political motivation (e.g. Anonymous)

Attack Surface

  • Every point a system can be attacked
  • Login pages, APIs, mobile apps, employees
  • Physical access (USB drops, tailgating)
  • Reduce attack surface: disable unused services
  • More features = larger attack surface (always a tradeoff)

Defense in Depth

  • Multiple layers of security controls
  • One breach doesn't compromise everything
  • Layers: perimeter → network → endpoint → data → user
  • A firewall failing doesn't mean data is exposed if MFA and encryption also exist
  • No single control is 100% effective

Week 1 Key Takeaway

  • Security is about risk management, not elimination. Every control has a cost.
  • CIA Triad is your mental model for evaluating any security decision.
  • Ask of any control: does it protect Confidentiality, Integrity, or Availability?
Week 2 — Common Attack Types
W2

Common Attack Types

Phishing · Malware · Social Engineering · DDoS

▼
Analogy: Attacks are like crime in the real world. Phishing = con artists tricking you into giving your wallet. Malware = someone secretly installing a tracking device on your car. DDoS = thousands of people blocking a shop entrance so no real customer can enter.

Phishing

  • Email/SMS tricks users into revealing credentials
  • Spear phishing — targeted, personalized, much more convincing
  • How to spot: urgency, bad grammar, suspicious links
  • Hover before clicking — check the real URL in status bar
  • 90% of breaches start with a phishing email

Malware Types

  • Virus — spreads by attaching to files
  • Trojan — disguised as legitimate software
  • Ransomware — encrypts your data, demands ransom
  • Spyware — silently steals information
  • Defenses: updated AV, EDR, regular patching, least privilege

Social Engineering

  • Manipulating people, not systems
  • Pretexting — fabricated scenario to gain trust
  • Vishing — voice phishing (phone calls)
  • Baiting — leaving infected USB drives around
  • Tailgating — following authorized person through door

DDoS (Distributed Denial of Service)

  • Floods a server with so much traffic it can't respond
  • Uses botnet armies of compromised machines
  • Directly attacks Availability in CIA Triad
  • Defenses: CDN, rate limiting, DDoS protection services
  • Famous victims: GitHub (2018 — 1.35 Tbps), AWS, Cloudflare

Week 2 Key Takeaway

  • 90% of breaches start with phishing. User training is the highest-ROI security investment.
  • Never click links in unexpected emails — go directly to the website by typing the URL.
  • Hover over any link before clicking to verify the real destination URL.
Week 3 — Network Security Basics
W3

Network Security Basics

Firewalls · VPN · Encryption · TLS & Certificates

▼

Firewalls

  • Filter traffic by IP address, port, and protocol
  • Stateless — checks each packet independently
  • Stateful — tracks active connections, smarter
  • Perimeter firewall = between internet and internal network
  • Host-based firewall = on the endpoint itself (Windows Firewall)

VPN (Virtual Private Network)

  • Encrypts traffic between your device and VPN server
  • Essential on public WiFi — prevents MITM attacks
  • IPsec — network-layer VPN, common in corporate
  • SSL/TLS VPN — browser-based, easier for remote access
  • VPN hides your traffic from the local network, not from internet

Encryption

  • Symmetric — same key for encrypt and decrypt (AES-256)
  • Asymmetric — public key encrypts, private key decrypts (RSA, ECC)
  • HTTPS uses both: asymmetric to exchange keys, then symmetric for speed
  • Encryption protects Confidentiality in the CIA Triad
  • Hashing (SHA-256) protects Integrity — one-way, not reversible

Certificates & TLS

  • Browser checks the website's digital certificate
  • Certificate is signed by a trusted Certificate Authority (CA)
  • Padlock icon = TLS encrypted connection
  • Padlock does NOT mean the site is safe — phishing sites use HTTPS too
  • TLS 1.3 is current standard; TLS 1.0/1.1 are deprecated

Week 3 Key Takeaway

  • Use HTTPS everywhere. Use a VPN on public WiFi.
  • These two habits protect against 80% of common network attacks.
  • A padlock confirms encryption — it does not confirm legitimacy.
Week 4 — Account Security & Safe Practices
W4

Account Security & Safe Practices

Password security · MFA · Least Privilege · Data Classification

▼

Password Security

  • Length beats complexity — 16+ characters minimum (NIST 2024)
  • Unique password for every site — never reuse
  • Password managers: Bitwarden (free, open source), 1Password
  • They generate cryptographically random passwords
  • They detect phishing by checking domain mismatch

Multi-Factor Authentication (MFA)

  • Something you know (password) + have (phone) + are (fingerprint)
  • TOTP apps (Google Authenticator, Aegis) beat SMS — SS7 attacks can steal SMS codes
  • MFA blocks 99.9% of automated account attacks (Microsoft, 2023)
  • Hardware keys (YubiKey) = strongest option for critical accounts
  • Enable MFA on every account that supports it, starting with email

Principle of Least Privilege

  • Grant only the minimum access needed for the job
  • No admin accounts for daily use — separate admin account
  • A database analyst needs SELECT, not DROP TABLE
  • Limits blast radius when an account is compromised
  • Review and remove unnecessary access regularly

Data Classification

  • Public — anyone can see (marketing materials)
  • Internal — employees only (internal wiki)
  • Confidential — restricted to specific teams (financials)
  • Restricted — highest sensitivity (passwords, keys, PII)
  • Different classification = different handling, storage, and access controls

Week 4 Key Takeaway

  • Enable MFA on every account that supports it.
  • Use a password manager. These two habits make you significantly harder to compromise than 95% of internet users.
  • Start today: install Bitwarden, enable MFA on your email account, check haveibeenpwned.com.
